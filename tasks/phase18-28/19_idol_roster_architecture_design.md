# Phase 19: アイドルロスター拡張設計

## 目的

アイドル10人以上を前提に、今後の追加方針をdocsへ整理する。
このPRでは新アイドルを実装しない。

## 作業

- `docs/planning/idol_roster_expansion_design.md` を更新する。
- 現在3人をプロローグ組として明記する。
- 追加アイドルのカテゴリを整理する。
- 全アイドルを全体生産倍率だけにしない方針を明記する。
- `passiveEffects` / `focusEffects` の導入方針を整理する。
- bond / 交流 / 個性の導入順を整理する。

## 追加検討

以下のようなロスター枠を仮設計する。
名前や確定設定は不要。役割だけでよい。

- 配信ブース系
- 記憶図書館系
- 地下駅跡 / 移動導線系
- 市場 / 広場系
- 修復工房系
- 記録保管区系
- 深層担当
- 転生後担当

## 禁止

- 新アイドルの実装
- 名前やビジュアルの正式決定
- 物語核心の確定
- 既存アイドルの効果変更

## 検証

```sh
npm run build
npm run check:content
npm run test
```
