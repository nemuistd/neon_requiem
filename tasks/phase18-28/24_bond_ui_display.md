# Phase 24: bond UI表示

## 目的

アイドル交流値をUI上で見えるようにする。

## 作業

### 注目アイドル枠

- 注目アイドルの `bond` を表示する。
- 表示名は「交流」を第一候補とする。
- 例: `交流 3`

### アイドルタブ

- 解放済みアイドルカードにbondを表示する。
- 未解放アイドルでは、bondより解放条件を優先する。

### 文言

- 「信頼」は使わない。
- 「個性」はまだ本格資源として扱わない。
- 交流値の説明は支援的・復興的にする。

## 禁止

- 詳細画面を実装しない。
- 新規大規模UIを作らない。
- `アイドルの様子` 追加はまだしない。

## 検証

```sh
npm run build
npm run check:content
npm run test
```

実ブラウザQAで確認:
- ライブ後に交流表示が増える。
- タブ切替後も表示される。
- リロード後も保存される。
