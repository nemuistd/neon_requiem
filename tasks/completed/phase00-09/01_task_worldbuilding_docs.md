# Codexタスク01: 世界観ドキュメントとREADMEの整合

## 目的

`docs/worldbuilding.md` を世界観正史として明確化し、READMEと今後の実装方針を揃えてください。

## 作業内容

1. `README.md` に `docs/worldbuilding.md` へのリンクを追加してください。
2. READMEの「既知の制限」または「今後の開発予定」に、以下の方針を短く追記してください。
   - 世界観・用語体系は `docs/worldbuilding.md` を基準にする。
   - 「聖歌」は通常システム名ではなく、危険な深層用語として扱う。
   - 今後は、歌アップグレードの通常名として「楽曲」「公演」「演目」「アップグレード」などを検討する。
   - 地下礼拝堂は削除せず、より深い進行段階へ再配置する候補とする。
   - 序盤では深層語を出しすぎず、表向きにはアイドル拠点運営・復興ゲームとして始める。
3. 既存文言にある「聖歌」が通常システム名に近い扱いになっていないか軽く確認し、README上の表現だけ必要なら調整してください。
4. まだコード変更は最小限にしてください。今回の主目的はドキュメント整備です。

## 禁止事項

- このタスクではゲームロジックを変更しない。
- 施設やアイドルの解放条件を変更しない。
- セーブ構造を変更しない。

## 完了条件

- `npm run build` が成功する。
- `npm run check:content` が成功する。
- 変更内容を要約する。
- 次タスクで必要な注意点があれば明記する。
