# 10 docs同期タスク

## 目的

README以外の既存docsに残っている旧記述を、現在の実装と世界観方針に同期する。

## 対象ファイル

- `docs/design_principles.md`
- `docs/neon_requiem_ui_principles.md`
- `docs/mvp_spec.md`
- `docs/concept.md`

## 前提

`docs/worldbuilding.md` を世界観正史とする。
未記載の設定は確定しない。
「聖歌」は通常システム名にしない。
「灯し手」は表向き呼称、「アンカー」は旧施設・祈念工学側の呼称。
地下礼拝堂は序盤施設ではなく、中盤以降の深層施設として扱う。

## 作業内容

### design_principles.md

以下を更新する。

- 「アイテムシステムはまだ未実装」を削除し、MVP実装済みとして記述する。
- 「灯り」を、現在の実装仕様では「灯るさ」に置換する。
- ただし演出表現としての「灯りが戻る」は残してよい。
- 現在の施設を以下に更新する。
  - 路地裏ステージ
  - ネオン掲示板
  - 地下礼拝堂
- 地下礼拝堂は中盤以降の深層施設として位置づける。
- saveVersion を 8 に更新する。
- 複数リソースは「完全実装済み」ではなく、`resources` 構造導入済み、現在の実資源は `tomorusa` と記述する。
- 意図的に未実装のものから「アイテムシステム」「設定画面」を外す。設定画面が最小実装で存在する場合は「設定画面の拡張」と書く。

### neon_requiem_ui_principles.md

以下を更新する。

- 「アイテムはまだ未実装です。」を削除し、アイテムタブMVP実装済みと記述する。
- 想定タブに「アイテム」を独立して含める。
- 上部ヘッダーの資源表示を「灯るさ」に更新する。
- 「地下礼拝堂」や「仮設礼拝ステージ」が序盤の通常UI前提として強く出すぎる箇所を緩和する。
- 深層語を序盤UIに出しすぎない方針を追記する。

### mvp_spec.md

この文書は「当初MVPの記録」と「現在の実装仕様」が混ざっているため、構成を整理する。

推奨構成:
- `この文書の位置づけ`
- `当初MVP v0.1`
- `現在の実装: Phase 3 基盤反映後`
- `現在の手動確認チェックリスト`
- `今後のスコープ外`

更新すべき内容:
- `saveVersion: 8`
- `resources.tomorusa`
- 表示名「灯るさ」
- 施設: 路地裏ステージ、ネオン掲示板、地下礼拝堂
- アイテムタブMVP実装済み
- 歌・アイテム・記録・アイドルタブを現在の状態へ合わせる
- 旧式の `lights`, `lightsPerSecond`, `offlineLights` は、当初MVPの記録として残す場合だけ明示する。現在仕様としては使わない。
- 手動確認チェックリストを現在の実装に合わせる。

### concept.md

以下を更新する。

- 「仮設礼拝ステージ」を序盤の確定名称として強く扱わない。必要なら「初期案では」と明記する。
- 「地下第七層・仮設礼拝ステージ」は初期案由来の表現として扱い、現在の初期施設は「路地裏ステージ」とする。
- 深月詩乃の肩書き「地下礼拝堂の聖歌係」は、現在の世界観方針と衝突しやすいため、将来改名候補として扱う。
- 第2施設は地下礼拝堂ではなく、現在はネオン掲示板であることを反映する。
- 地下礼拝堂は中盤以降の深層施設として扱う。
- 初期リソースは「灯るさ」に更新する。
- 「灯り」は演出語として残してよい。

## 禁止事項

- 新しい重大世界観設定を勝手に追加しない。
- 実装コードは変更しない。
- 施設やアイドルの解放条件は変更しない。

## 完了条件

- docs全体で、現在仕様と明確に矛盾する旧記述がなくなる。
- README / docs/worldbuilding.md と矛盾しない。
- `npm run build`
- `npm run check:content`
