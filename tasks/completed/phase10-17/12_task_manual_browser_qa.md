# 12 ブラウザ手動QAタスク

## 目的

ビルドとコンテンツチェックだけでは確認できない、実ブラウザ上の基本操作を確認し、結果を記録する。

## 作業内容

### 1. QA記録ファイルを追加

`docs/qa_manual_checklist.md` を追加または更新する。

以下を含める。

- 実行日
- 実行環境
- 実行したコマンド
- 確認項目
- 結果
- 発見した問題
- 修正済み/未修正

### 2. 開発サーバーで確認

```sh
npm install
npm run dev
```

または環境に応じて既存依存を使う。

ブラウザで以下を確認する。

#### 初期状態

- 新規セーブで起動できる。
- 表示資源が「灯るさ」になっている。
- 灯るさが0から始まる。
- 毎秒が0.00 / 秒から始まる。
- 注目アイドルに音羽 灯里が表示される。
- 復興区画タブが初期表示される。

#### ライブ

- 「ライブする」を押すと灯るさが増える。
- メッセージが更新される。
- リロード後も増えた灯るさが保存される。

#### 施設

- 灯るさ10で路地裏ステージを強化できる。
- 強化で灯るさが消費される。
- レベルが上がる。
- 毎秒生産が増える。
- 時間経過で灯るさが増える。

#### ネオン掲示板

- 路地裏ステージLv10到達前は未解放表示になる。
- 路地裏ステージLv10で解放される。
- ネオン掲示板を強化できる。

#### 地下礼拝堂

- ネオン掲示板Lv10到達前は未解放表示になる。
- 解放条件表示が正しい。
- 序盤2施設目として出ていないことを確認する。

#### 歌

- 歌タブを開ける。
- 条件を満たすと歌が購入可能になる。
- 購入後、効果が反映される。
- 購入済み表示が保存される。

#### アイテム

- アイテムタブを開ける。
- 条件を満たすとアイテムが購入可能になる。
- 購入後、効果が反映される。
- 購入済み表示が保存される。

#### アイドル

- アイドルタブを開ける。
- 未解放アイドルの解放条件が見える。
- 解放後に注目アイドルへ切り替えられる。
- リロード後も注目アイドルが保存される場合は保存を確認する。保存されない仕様なら、その旨をdocsへ記録する。

#### 記録

- 記録タブを開ける。
- 初期解放記録が読める。
- 読むと既読になる。
- リロード後も既読が残る。
- 記録を読んでも灯るさや生産量は変わらない。

#### オフライン報酬

- デバッグAPIまたはlocalStorage操作で `lastSavedAt` を過去にし、オフライン報酬が加算されることを確認する。
- 最大12時間上限が働くことを確認する。

#### 設定・デバッグ

- 設定ボタンを開ける。
- セーブ削除が動く。
- `window.__NEON_DEBUG__.resetSave()` が動く。
- `window.__NEON_DEBUG__.setSaveForTest()` が動く。

## 完了条件

- `docs/qa_manual_checklist.md` に結果が記録される。
- 重大な不具合があれば同PRで修正するか、別Issue/残課題として明記する。
- `npm run build`
- `npm run check:content`
