# 33_ch9_convergence

2026-05-25 実装済み。関連仕様、手動QA、product / planning docs も Ch.9 収束補強の最小縦切りとして同期済み。

## 目的

Ch.9 起点の `再観測拠点`、`名前のない劇場`、`白霧 燐` に続き、Ch.9 収束補強を最小実装する。

バインダー封書の全文解読、名前のない劇場での残響、再び廻る / 続ける導線を扱う。ただし、主人公の正体、霞の意志、旧計画の全貌を単一の答えとして確定しない。

## 実装範囲

- 歌 `theLastName` / 「最後の名前」。
- 記録「バインダー封書・全文」。
- 記録「名前のない劇場・残響公演」。
- Ch.9 収束条件到達後の廻タブ導線「再び灯るものへ」。
- 関連する current spec、manual QA、product / planning docs の同期。

## 解放条件

- 「最後の名前」: 廻2以上 + 名前のない劇場 Lv1。
- 「バインダー封書・全文」: 廻2以上 + 名前のない劇場 Lv3 + 「最後の名前」取得。
- 「名前のない劇場・残響公演」: 廻2以上 + 名前のない劇場 Lv1 + 「最後の名前」取得。
- 「再び灯るものへ」導線: 廻2以上 + 名前のない劇場 Lv3 + 「最後の名前」取得。

## 受け入れ条件

- 廻1では Ch.9 収束要素が通常 UI に表示されない。
- 廻2以降、条件を満たすと「最後の名前」を取得できる。
- 「最後の名前」取得後、施設生産倍率と記憶断片の獲得見込みが増える。
- 条件到達後、バインダー封書・全文と残響公演の記録が読める。
- 廻タブに、復興を続ける / もう一度廻る導線が表示される。
- 新しい章保存フィールドや `SAVE_VERSION` を増やさない。
- 物語上の解決や核心設定の確定をしない。

## 確認

- `npm.cmd run test`
- `npm.cmd run check:content`
- `npm.cmd run build`
- 必要に応じて、実ブラウザで Ch.9 収束導線とモバイル幅を確認する。
