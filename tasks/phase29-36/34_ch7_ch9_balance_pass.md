# 34_ch7_ch9_balance_pass

## 目的

Ch.7〜Ch.9 収束補強までの最小縦切りを通し確認し、廻後の進行速度と数値バランスを一次調整する。

このタスクは `P-M9-PASS-26` に対応する。新しい章、施設、アイドル、物語核心の確定は扱わない。

## 実施範囲

- 初回廻から Ch.7 入口までの再進行速度確認。
- Ch.7 から Ch.8、Ch.9 起点、Ch.9 収束条件までの到達感確認。
- 記憶断片獲得量、廻後バフ価格、廻後基礎加速、主要施設コスト、歌コストの一次調整候補整理。
- 必要な場合のみ、小さな数値調整を実装する。
- `docs/current_spec.md`、`docs/manual_qa_checklist.md`、product / planning docs の同期。

## 受け入れ条件

- 廻後のプレビュー、実付与、次断片ゲージが同じ計算方針のまま崩れていない。
- Ch.7、Ch.8、Ch.9 起点、Ch.9 収束補強の解放順が変わっていない。
- 数値調整を行う場合は、変更理由と対象値がドキュメントに残っている。
- 主人公の正体、霞の意志、旧計画の全貌を確定しない。
- 新しい保存フィールドや `SAVE_VERSION` を増やさない。

## 確認

- `npm.cmd run test`
- `npm.cmd run check:content`
- `npm.cmd run build`
- 必要に応じて、実ブラウザで廻後から Ch.9 収束導線までの代表状態を確認する。
