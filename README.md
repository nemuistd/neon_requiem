# NEON REQUIEM

NEON REQUIEM は、地下都市・ネオン・聖歌・アイドルをモチーフにした、ブラウザ向けの静的インクリメンタルゲームです。

プレイヤーは、地下都市の路地裏で歌う初期アイドル「音羽 灯里」を支え、ライブによって拍手を集め、壊れかけた「路地裏ステージ」を少しずつ復興していきます。

現在は MVP v0.1 として、最小のゲームループだけを実装しています。

## 実装済みの機能

- 「ライブする」ボタンによる手動拍手獲得
- 基本リソース「拍手」
- 初期アイドル「音羽 灯里」の表示
- 初期施設「路地裏ステージ」の表示
- 拍手を消費した路地裏ステージ強化
- 強化コスト計算: `floor(10 * 1.15 ^ level)`
- 自動生産: `路地裏ステージ Lv * 0.1 拍手 / 秒`
- 経過時間に基づく自動生産
- `localStorage` による保存・読み込み
- 最終保存時刻に基づく簡易オフライン報酬
- Vite + TypeScript による静的 Web アプリ構成
- GitHub Pages 公開を見据えた相対パスビルド設定

## ローカル起動方法

依存関係をインストールします。

```sh
npm install
```

開発サーバーを起動します。

```sh
npm run dev
```

表示されたローカル URL をブラウザで開いてください。通常は以下です。

```text
http://localhost:5173/
```

## ビルド方法

本番用ファイルを生成します。

```sh
npm run build
```

生成物は `dist/` に出力されます。

ビルド結果をローカルで確認する場合は、以下を実行します。

```sh
npm run preview
```

## 手動確認チェックリスト

- アプリがローカルで起動する。
- 新規セーブで拍手が 0 から始まる。
- 「ライブする」を押すと拍手が 1 増える。
- 拍手 10 で路地裏ステージを強化できる。
- 強化時に拍手が消費される。
- 路地裏ステージのレベルが上がる。
- 強化後、秒間拍手量が増える。
- クリックしなくても時間経過で拍手が増える。
- ページ再読み込み後も拍手とステージ Lv が残る。
- 離れていた時間に応じてオフライン拍手が加算される。
- スコープ外の機能が追加されていない。

## 既知の制限

- MVP v0.1 は完成版ゲームではありません。
- アイドルは「音羽 灯里」1人のみです。
- 施設は「路地裏ステージ」1つのみです。
- リソースは「拍手」1種類のみです。
- オフライン報酬に上限はありません。
- セーブデータ破損時の詳細な復旧 UI はありません。
- セーブデータのリセット UI はありません。
- 音声、画像、立ち絵、ストーリーシステムは未実装です。
- モバイル向けの本格的な最適化はまだ行っていません。

## GitHub Pages について

将来的に GitHub Pages で公開する予定です。

現在の Vite 設定では `base: "./"` を使っており、リポジトリ名が変わってもビルド後のアセットを相対パスで読み込めるようにしています。
