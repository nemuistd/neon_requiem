# NEON REQUIEM 手動確認チェックリスト

## 基本

- アプリがローカルで起動する。
- `npm run build` が成功する。
- `npm run check:content` が成功する。
- 新規セーブで灯るさが 0 から始まる。
- Console の `window.__NEON_DEBUG__.resetSave()` で初期状態に戻せる。
- 設定画面からセーブを削除できる。

## 初回3分導線

- 新規セーブで起動後、最初に「灯るさ」と「ライブする」ボタンが視認できる。
- 起動直後のメッセージで、路地裏ステージをもう一度人が来られる場所に戻す雰囲気が伝わる。
- 「ライブする」を1回押すと、灯るさが増え、灯里の歌によって路地に灯るさが戻ったことが分かるメッセージが出る。
- 灯るさ不足の状態で強化や購入を試すと、次にライブで灯るさを戻せばよいことが分かる。
- 灯るさ10到達後、路地裏ステージを強化できる。
- 路地裏ステージ強化後、灯るさが消費され、毎秒生産が0より大きくなる。
- 30秒ほど何も押さなくても、毎秒生産で灯るさが増える。
- 初回3分の範囲で、祈念工学、アンカー、祈念負荷、聖歌を知らなくても目標が分かる。
- 初回3分の範囲で、プレイヤーがアイドルを所有・搾取しているように見える文言がない。

## ライブ / 生産 / 保存

- 「ライブする」を押すと灯るさが 1 増える。
- 灯るさ 10 で路地裏ステージを強化できる。
- 施設強化時に灯るさが消費される。
- 施設強化後、秒間灯るさ量が増える。
- クリックしなくても時間経過で灯るさが増える。
- ページ再読み込み後も灯るさと施設 Lv が残る。
- 離れていた時間に応じてオフライン灯るさが加算される。

## 施設進行

- 路地裏ステージ Lv10 到達前は、ネオン掲示板が未解放として表示される。
- 路地裏ステージ Lv10 到達後、ネオン掲示板が解放される。
- 地下広場 Lv4 到達後、名前の記録壁が解放される。
- 名前の記録壁 Lv3 到達前は、地下礼拝堂が未解放として表示される。
- 名前の記録壁 Lv3 到達後、地下礼拝堂が解放される。
- 地下礼拝堂が序盤2施設目として表示されない。

## アイドル

- 初期注目アイドルは音羽 灯里。
- 未解放アイドルに解放条件が表示される。
- ネオン掲示板 Lv5 到達前は、朝霧 結が未解放として表示される。
- ネオン掲示板 Lv5 到達後、朝霧 結が解放される。
- 朝霧 結を注目アイドルに切り替えられる。
- 地下礼拝堂 Lv3 到達前は、深月 詩乃が未解放として表示される。
- 地下礼拝堂 Lv3 到達後、深月 詩乃が解放される。
- 深月 詩乃を注目アイドルに切り替えられる。
- リロード後も選択可能な注目アイドルが保存される。

## 歌

- 歌タブを開ける。
- 路地裏ステージ Lv5 かつ 80 灯るさで「路地裏のイントロ」を取得できる。
- 「路地裏のイントロ」取得後、ライブ1回の獲得灯るさが 2 になる。
- 地下礼拝堂 Lv1 かつ 450 灯るさで「礼拝堂のハーモニー」を取得できる。
- 地下礼拝堂 Lv5 かつ 1800 灯るさで「薄明のコーラス」を取得できる。
- 生産倍率系の歌を取得すると秒間灯るさ量が増える。
- 購入済み歌がリロード後も購入済みとして残る。

## アイテム

- アイテムタブを開ける。
- 路地裏ステージ Lv3 かつ 100 灯るさで「古いネオン管」を購入できる。
- 路地裏ステージ Lv4 かつ 140 灯るさで「手書きの告知ポスター」を購入できる。
- 「手書きの告知ポスター」購入後、ライブ1回の獲得灯るさがさらに 1 増える。
- 路地裏ステージ Lv5 かつ 220 灯るさで「半券の束」を購入できる。
- 「半券の束」購入後、ライブ1回の獲得灯るさがさらに 1 増える。
- 路地裏ステージ Lv6 かつ 260 灯るさで「携帯スポットライト」を購入できる。
- 「携帯スポットライト」購入後、毎秒灯るさの表示が増える。
- ネオン掲示板 Lv2 かつ 500 灯るさで「交代用連絡ボード」を購入できる。
- 「交代用連絡ボード」購入後、オフライン報酬は基礎50%に `x1.10` が乗り、同じ時間のオンライン自然生産の55%になる。
- ネオン掲示板 Lv3 かつ 600 灯るさで「録音済みの短い挨拶」を購入できる。
- 購入済みアイテムがリロード後も購入済みとして残る。

## 記録

- 記録タブを開ける。
- 初期解放の記録が読める。
- 施設Lvや歌取得に応じて記録が解放される。
- 記録を読むと既読になる。
- 既読状態がリロード後も残る。
- 記録を読んでも灯るさや生産量は変化しない。
- surface の記録に、祈念工学、アンカー、聖歌、祈念負荷が出ていない。

## UI / 用語

- タブ切替が動く。
- 主要ボタンの disabled 状態が正しい。
- 数値リソース表示が「灯るさ」になっている。
- 「聖歌」が通常システム名になっていない。
- 序盤UIに祈念工学、アンカー、祈念負荷、聖歌などが出すぎていない。
- 地下礼拝堂が削除されていない。
- メッセージ文言が不自然でない。
- モバイル幅で大きく崩れていない。
