# NEON REQUIEM 世界観・用語体系

## 目的

この文書は、NEON REQUIEM の世界観と用語体系を整理し、今後のアイドル能力、施設、資源、メッセージ文言を設計するための基準にする。

序盤のゲームは、表向きには「荒廃した街でアイドル拠点を運営し、ライブや施設復興によって人々を集めていく放置・育成ゲーム」として始める。

ただし、世界の裏側には異常現象がある。世界は、人々の認知の平均に適応しようとする傾向を持っている。多くの人に見られ、覚えられ、名前を呼ばれているものは安定する。逆に、忘れられたもの、見捨てられたもの、意味を失ったものは、現実の中で輪郭を失っていく。

序盤では、この現象を「忘れ去られた施設は廃れる」「人が集まらない街は寂れる」といった一般的な社会現象のように見せる。物語が進むにつれて、それが比喩ではなく、実際の物理現象であることが明らかになる。

世界は忘却に対して非常に脆弱であり、人々の注意もまた世界の不安定さに応じて散漫になっている。そのため、忘却と不安定化が悪循環を起こしている。

## 霞

外的現象の名前は「霞」とする。

霞は単なる気象現象ではない。人々の忘却、認知の揺らぎ、名前の喪失、意味の曖昧化に呼応して、世界の輪郭を曖昧にする外的現象である。

霞そのものに明確な人格や意志があるかどうかは、現時点では断定しない。神というより、自然災害に近い外的現象として扱う。

霞が濃くなると、場所、記憶、名前、役割、施設、人間関係などが不安定になる。誰にも覚えられていない場所は霞に沈みやすい。名前を失ったものは輪郭を失いやすい。

## 祈念工学

旧時代の技術体系の名前は「祈念工学」とする。

祈念工学とは、人々の願い、祈り、期待、注目、信仰に近い感情を束ね、現実を望ましい形へ固定または改変しようとした技術体系である。

かつて人類は、世界が認知に影響される性質を発見し、それを利用しようとした。一部の権力者や研究者たちは、祈念工学を復興、統治、理想都市建設、戦争、記憶保存、死者蘇生などに利用しようとした。

しかし、祈念工学は人類が完全に制御できる技術ではなかった。実際には、祈念工学は霞を介して外部の応答を招く方法だった。

大規模な祈念工学実験の失敗により、世界は現在のような不安定な状態に陥った。

## 主人公

主人公は記憶を封印された状態にある。

主人公が過去の自分と完全に同一の存在かどうかは、現時点では保留する。記憶を失った本人なのか、記憶と権限だけを継いだ代行存在なのか、別の存在なのかは、後から明かせる余地を残す。

主人公の記憶が封印されている理由は、記憶を持ち続けていること自体が霞の呼応を招きやすいためである。この世界では、強く覚えていること、強く願うこと、強く名付けることが現実に影響してしまう。そのため、旧計画の核心を知る主人公の記憶は危険な媒介になりうる。

主人公は、なぜ自分がアイドル拠点を運営しているのかを完全には理解していない。残されたマニュアル、施設ログ、断片的な記憶に従い、復興を進めている。

## 灯し手

アイドル適性者の呼び名は「灯し手」とする。

灯し手とは、人々の注意、応援、期待、祈りに似た感情を集め、それを歌、公演、物語、キャラクターとして安全に変換できる者たちである。

この世界では、人々の注意を集める存在は、世界を安定させる焦点になりうる。アイドルはまさに、人々に名前を呼ばれ、顔を覚えられ、期待され、繰り返し見られる存在である。そのため、灯し手は霞に沈みかけた街や施設を安定させるための重要な存在になる。

ただし、灯し手は神ではない。灯し手を神格化してはならない。

## アンカー

祈念工学者や旧施設の管理文書では、灯し手は「アンカー」と呼ばれる。

「灯し手」は表向き・人間的・物語的な呼称。「アンカー」は工学的・管理的・非人間的な呼称である。

旧ログでは、以下のような語彙を使ってよい。

- アンカー個体
- アンカー適性
- 認知固定率
- 霞曝露耐性
- 祈念負荷
- 自己輪郭の維持
- 観測者数
- 局所安定化

## 聖歌

「聖歌」はメインシステム名にはしない。通常のライブや歌とは異なる、危険な深層用語として扱う。

聖歌とは、祈念工学において、集めた祈念を霞へ直接届けてしまう危険な形式、または音声プロトコルである。

通常の歌やライブは、人々の注意を安全に集め、灯し手を中心に街や施設を安定させる。一方で聖歌は、集めた祈念を霞へ直接接続してしまう。

世界観上の重要な原則は以下とする。

- 灯し手は、神ではない。
- 公演は、儀式ではない。
- 歌は、聖歌ではない。

少なくとも、そう扱い続けなければならない。

既存ドキュメントやUI案には「聖歌」が通常システム名に近い形で残っている箇所がある。今回の整理以降は、将来的に「楽曲」「公演」「演目」「アップグレード」などへ置き換える候補として扱う。

## 表向きの用語と裏側の意味

表向きのUIや序盤の説明では、あまり不穏な言葉を出しすぎない。序盤は、普通のアイドル復興ゲームに見えるようにする。

ただし、ログや一部メッセージにだけ異常性を混ぜていく。

| 表向きの用語 | 裏側の意味 |
| --- | --- |
| ファン | 観客・応援者。裏側では観測者。 |
| 応援 | 支援感情。裏側では弱い祈念。 |
| ライブ / 公演 | アイドル活動。裏側では注意を集める安定化手段。 |
| 施設復興 | 街の再建。裏側では局所現実の再固定。 |
| 灯るさ | 街や施設が霞に抗って安定している度合い。明るさではなく、この世界固有の安定度資源。 |
| 個性 | 灯し手本人の自己輪郭・キャラクター性・自分らしさを保つ力。 |
| 霞 | 忘却や認知の揺らぎに呼応して世界を曖昧にする外的現象。 |
| 祈念 | 願い、祈り、期待、注目、信仰に近い感情の総称。序盤では前面に出しすぎない。 |

## 資源名の暫定方針

今後の実装で使う資源名は、以下を基本候補とする。

- ファン
- 応援
- 灯るさ
- 記憶
- 個性

「灯るさ」は、街や施設の安定度・復興度・霞への抵抗力を表す。

「個性」は、アイドル本人の自己同一性やキャラクター性を保つ力を表す。

「信頼」を自己同一性維持の資源として使う案は採用しない。信頼は人間関係やファンとの関係値として使う可能性はあるが、灯し手本人の輪郭を保つ力としては「個性」の方が適切である。

## 開示の順番

### 序盤

- 荒廃した街でアイドル活動を始める。
- 人が集まると街に灯りが戻る。
- 忘れられた場所は廃れていく、という程度に見せる。
- 「霞」は自然現象または雰囲気のように見せる。

### 中盤

- ファン数や公演が、施設の安定に直接影響していることが分かる。
- ログに「観測者」「アンカー」「祈念負荷」などの語が混ざる。
- 「忘却」は比喩ではなく物理現象だと示す。

### 後半

- 祈念工学の存在が明らかになる。
- かつて人類が願いによる現実改変を利用しようとしていたことが分かる。
- その技術が霞を呼び込むものだったと分かる。
- 主人公の記憶封印の理由が明らかになっていく。
- 聖歌が危険な深層用語として登場する。

## 実装上の注意

この段階では、世界観設定は主に今後の文言・資源・能力設計の基準として扱う。

- 既存コードの大規模変更は行わない。
- 序盤UIでは、祈念工学・アンカー・祈念負荷などの裏側用語を前面に出しすぎない。
- 「聖歌」が通常システム名として使われている場合、将来的には別語に置き換える候補にする。
- 「信頼」が自己同一性維持の資源として使われている場合、将来的には「個性」に置き換える候補にする。
- 安定度・復興度に相当する資源や表示がある場合、「灯るさ」という名称の採用を検討する。
- アイドル適性者の表向き名称は「灯し手」、旧工学者側の呼称は「アンカー」とする。
- ゲーム体験の中心は搾取や絶望ではなく、回復、居場所づくり、復興に置く。
