# NEON REQUIEM UI設計方針

この文書は、NEON REQUIEM のUIを今後拡張していく際に、画面構成・情報優先度・キャラクター表示の方針がぶれないようにするための設計原則をまとめたものです。

これは最終仕様書ではありません。実装後のプレイ感、画面サイズ、追加要素の量に応じて更新していきます。

---

# 1. 基本方針

NEON REQUIEM は、単なる数値増加ゲームではなく、地下都市・ネオン・歌・アイドルを軸にした美少女インクリメンタルゲームです。

そのため、UIでは以下を両立します。

- インクリメンタルゲームとして必要な数値・操作・強化の見やすさ
- 美少女アイドルゲームとしてのキャラクター存在感
- 地下都市、ネオン、歌、少しダークな雰囲気
- 将来の施設・アップグレード追加に耐える拡張性
- メッセージによる世界観表現と進行フィードバック

画面は「綺麗だが操作しづらい」方向にも、「便利だがキャラの存在感や世界観が消える」方向にも寄せすぎないことを目指します。

---

# 2. 改定後の大きなレイアウト方針

NEON REQUIEM の基本レイアウトは、以下の4層で考えます。

```text
[1] 上部ヘッダー / 操作パネル
[2] メッセージバー
[3] メイン領域
    左: 注目アイドル枠
    右: タブ切り替え式コンテンツ領域
[4] 必要に応じた補助領域
```

重要なのは、施設やアップグレードをすべて同じ縦長リストに混ぜないことです。

今後要素が増えるため、右側のメインコンテンツ領域は、最終的にはタブまたはカテゴリ切り替え式にします。

想定タブ:

- 復興区画
- 歌 / アイテム
- アイドル
- 記録

現在の実装では、右側にタブ見出しを表示し、「復興区画」「歌」「記録」を切り替えて表示しています。

アイドルタブの内容はまだ未実装です。

---

# 3. スクロールに対する考え方

スクロール自体は許容します。

施設、アップグレード、未解放要素、ログ、詳細説明などは将来的に増えるため、すべてを1画面に収めようとするとUIが破綻します。

ただし、スクロールによって以下が見えなくなる状態は避けます。

- 現在の主要リソース
- 秒間生産量
- 主要操作ボタン
- 重要な最新メッセージ
- 少なくとも1人の注目アイドル

目標は「スクロールをなくすこと」ではなく、「スクロールしてもゲームの顔と重要情報を見失わないこと」です。

スクロールしてよい領域:

- 復興区画のカード一覧
- 歌 / アイテム一覧
- 未解放要素一覧
- 詳細説明
- 過去ログ

見失わせない領域:

- 上部ヘッダー
- 主要リソース
- ライブするボタン
- メッセージバー
- 注目アイドル枠

---

# 4. 上部ヘッダー / 操作パネル

上部ヘッダーは、ゲームを操作するための最重要領域です。

表示するもの:

- コンパクトなタイトル: NEON REQUIEM
- 短いサブタイトル
- 灯り
- 毎秒生産量
- ライブするボタン
- 設定ボタン

## 4.1 タイトルの扱い

現在の大きなタイトル表示は、初見の印象作りとしては有効です。

しかし、プレイ画面として常時大きく表示すると、リソースや操作、施設表示の領域を圧迫します。

今後は以下の方針とします。

- タイトルは消さない
- ただしプレイ画面ではコンパクトにする
- 上部ヘッダー内にブランドとして残す
- サブタイトルは短くする

例:

```text
NEON REQUIEM
地下都市復興ライブ
```

または

```text
NEON REQUIEM
地下歌都市・仮設礼拝ステージ
```

## 4.2 設定ボタン

設定ボタンは1か所に統一します。

原則として、デスクトップでは右上に1つだけ配置します。

左下などに重複して設定ボタンを置かないこと。

---

# 5. メッセージバー

メッセージは NEON REQUIEM において重要な世界観表現の手段です。

単なる通知ではなく、以下を担います。

- オートセーブなどの状態通知
- 施設強化の反応
- オフライン報酬の報告
- 新要素解放の演出
- 地下都市やアイドルの短いフレーバーテキスト
- プレイヤーへの語りかけ

そのため、メッセージは小さく右端に押し込めない方針とします。

## 5.1 配置

メッセージバーは、上部ヘッダーの直下に横長で配置します。

推奨:

```text
[上部ヘッダー]
[横長のメッセージバー]
[メイン領域]
```

中央上部に近い位置で、プレイヤーの視線に自然に入るようにします。

## 5.2 表示内容

常時表示するのは最新1件を基本とします。

例:

```text
最新メッセージ: 灯里の歌声が路地裏に響き、ひび割れた街路に小さな光が戻りました。
```

または、強化や報酬の表現:

```text
最新メッセージ: 地下礼拝堂を強化しました。-100 灯り
```

オートセーブ成功は頻繁に発生するため、現在の実装ではメインメッセージを上書きしない。

## 5.3 サイズ

メッセージバーには十分な横幅を与えます。

短い通知だけでなく、やや長めの世界観メッセージも入るようにします。

ただし、長文を常時表示しすぎてメイン画面を圧迫しないようにします。

## 5.4 過去ログ

過去ログは常時表示しなくてよいです。

将来的に必要になった場合は、以下のように扱います。

- 記録タブ
- 通知履歴
- 折りたたみログ
- 下部補助ログ

MVP後しばらくは、最新1件表示で十分です。

---

# 6. メイン領域

メイン領域は、左の注目アイドル枠と、右のタブ切り替え式コンテンツ領域で構成します。

```text
┌───────────────┬──────────────────────────┐
│ 注目アイドル      │ タブ切り替え式コンテンツ領域       │
│ 音羽 灯里        │ 復興区画 / 歌 / アイドル / 記録   │
│ 立ち絵/カード     │ 選択中タブの内容                  │
└───────────────┴──────────────────────────┘
```

---

# 7. 注目アイドル枠

## 7.1 基本方針

常に表示するのは「現在の注目アイドル」1人を基本とします。

注目アイドルは、画面の顔として大きめに表示します。

全アイドルを常時表示しようとすると、1人あたりの表示が小さくなり、キャラクター性が薄くなります。そのため、全員常時表示は避けます。

## 7.2 表示内容

注目アイドル枠は、単なる雰囲気カードではなく、意味のある情報枠にします。

表示内容の例:

- 名前
- ふりがな
- 肩書き
- 短いフレーバーテキスト
- パッシブ効果
- 現在のボーナスまたは関係効果
- 詳細ボタン
- 将来的には立ち絵またはポートレート

## 7.3 初期状態

初期の注目アイドルは **音羽 灯里（おとは あかり）** とします。

灯里は初期ステージ、初期ライブ、灯り、路地裏ステージに関係するゲームの看板役です。

## 7.4 将来的な切り替え方針

将来的には、選択中の施設や区画に対応するアイドルを注目表示する方針が望ましいです。

例:

- 路地裏ステージを見ているとき: 音羽 灯里
- 地下礼拝堂を見ているとき: 深月 詩乃
- ネオン発電区を見ているとき: その区画に対応するアイドル

これにより、施設・区画とアイドルの関係が画面上で自然に伝わります。

現在の実装では、注目アイドル枠の下に小さなアイドル切替リストを表示しています。

未解放アイドルは候補として表示し、解放条件も表示します。ただし、未解放状態では選択できません。

注目アイドルの選択状態はまだ保存していません。リロード後は音羽 灯里に戻ります。

---

# 8. 右側コンテンツ領域

右側コンテンツ領域は、将来的に要素が増えることを前提に、タブまたはカテゴリ切り替え式で設計します。

## 8.1 基本タブ

想定する主要タブ:

### 復興区画

施設・区画を表示します。

例:

- 路地裏ステージ
- 地下礼拝堂
- ネオン発電区
- 記憶図書館

### 歌 / アイテム

倍率効果や特殊効果を表示します。

現在の歌:

- 路地裏のイントロ
- 礼拝堂のハーモニー
- 薄明のコーラス

アイテムはまだ未実装です。

### アイドル

解放済みアイドル、未解放アイドル、各アイドルの概要を表示します。

ただし、全アイドルを常時メイン画面に大きく並べるのではなく、このタブで一覧管理します。

### 記録

世界観ログ、進行記録、主人公の断片的な記憶を扱います。

現在の実装では、進行条件で解放された記録を読めます。記録には数値効果を持たせず、既読状態だけを保存します。

メッセージバーの過去履歴も、将来的には記録タブにまとめられます。

## 8.2 初期段階での扱い

初期段階では、すべてのタブを完全実装する必要はありません。

ただし、少なくとも以下の分類を意識します。

- 復興区画とアップグレードを混ぜすぎない
- アイドル情報を施設一覧に埋もれさせない
- ログやメッセージ履歴を施設カードの末尾に混ぜない

## 8.3 スクロール

右側コンテンツ領域はスクロールしてよい領域です。

ただし、タブやセクション見出しは見失いにくくします。

---

# 9. 施設・アップグレード一覧の扱い

## 9.1 施設カード

施設カードには、最低限以下を表示します。

- 施設名
- 短い説明
- Lv
- 強化コスト
- 生産量または効果
- 強化ボタン

## 9.2 アップグレードカード

アップグレードカードには、最低限以下を表示します。

- アップグレード名
- 効果説明
- Lv または購入状態
- コスト
- 強化または購入ボタン

## 9.3 未解放カード

未解放要素は、すべて隠すのではなく、適度に存在を示してよいです。

例:

```text
信徒ランク Lv.20 で解放
```

ただし、未解放要素を出しすぎると圧迫感が出るため、序盤は控えめにします。

---

# 10. 左下メニューに対する方針

デスクトップ画面では、左下に独立したメニューを置く方針は現時点では採用しません。

理由:

- 注目アイドル枠の一部なのか、全体ナビゲーションなのかが分かりにくい
- 右上設定ボタンと重複しやすい
- 画面構造が複雑に見える
- 右側コンテンツ領域のタブと役割が競合する

画面切り替えやカテゴリ移動は、右側コンテンツ領域のタブで扱う方針とします。

設定は右上に1つだけ置きます。

---

# 11. 美少女要素の表示方針

NEON REQUIEMでは、美少女アイドル要素を数値ゲームの装飾にしないことが重要です。

方針:

- 常に少なくとも1人のアイドルを表示する
- 注目アイドルは大きめに表示する
- 全員を常時小さく並べることは避ける
- 施設やアップグレードとアイドルの関係を持たせる
- 将来的に立ち絵を追加できる余白を残す

現時点では画像なしでもよいですが、後から立ち絵を置けるように、注目アイドル枠には十分な面積を確保します。

---

# 12. 操作性と雰囲気のバランス

NEON REQUIEMのUIでは、雰囲気作りも重要です。

望ましい印象:

- 暗い地下都市
- ネオンの光
- 歌、礼拝堂、ステージ
- 少しダークだが美しい
- アイドルの存在感

ただし、雰囲気演出が以下を妨げてはいけません。

- 文字の読みやすさ
- ボタンの押しやすさ
- コストや生産量の確認
- 強化可能かどうかの判断
- 最新メッセージの視認性

原則として、演出より操作性を優先します。操作性を確保したうえで、色・枠線・光・背景・短いテキストで雰囲気を出します。

---

# 13. スマホ幅の最低方針

スマホ幅では、左右分割ではなく縦積みを基本とします。

優先順:

1. 上部ヘッダー / 操作パネル
2. メッセージバー
3. 注目アイドル枠
4. タブ切り替え式コンテンツ領域

スマホでは常にすべてを画面内に収めることは難しいため、少なくとも上部操作、メッセージ、注目アイドルが見つけやすい構成にします。

---

# 14. 現時点でのUI改善方針

現在の実装から次に改善する場合は、以下を優先します。

## 14.1 実装済みの改善

- 大きすぎるタイトル領域をコンパクトにする
- 灯り、毎秒、ライブするボタン、設定ボタンを上部ヘッダーにまとめる
- メッセージを上部ヘッダー直下の横長バーに移動する
- メッセージバーに十分な横幅を与える
- 左下メニューは採用しない
- 設定ボタンは右上に統一する
- 注目アイドル枠を用意する
- 注目アイドル枠には、名前、肩書き、パッシブ効果、ボーナス、詳細ボタンなどを表示する
- 右側コンテンツ領域に、復興区画 / 歌 / アイドル / 記録のようなタブ構造を想定する
- 初期実装では復興区画タブだけでもよいが、将来の分類を見越した構造にする
- アイドル切替候補を表示する
- 未解放アイドルに解放条件を表示する

## 14.2 まだ行わない改善

- 注目アイドル選択状態の保存
- 全タブの完全実装
- 立ち絵制作や画像追加
- スマホ専用の細かい最適化
- 詳細な過去ログ機能
- 複雑なアニメーション付きヘッダー
- アイドル詳細画面
- アイドルごとの実効果

---

# 15. Codexへの実装時注意

CodexがUIを変更する場合は、以下を守ること。

- ゲームロジック、保存処理、数値計算を変更しない
- UI再配置とCSS調整を主目的にする
- 新機能を不用意に追加しない
- 施設、アップグレード、アイドル、ログの役割を混ぜない
- 常に少なくとも1人の注目アイドルが見える構成を維持する
- リソース、毎秒、ライブするボタン、メッセージバーを見失いにくい位置に置く
- 画面が数値カードだけにならないようにする
- 設定ボタンを複数箇所に置かない
- 左下メニューを安易に追加しない
- 右側コンテンツは、将来的なタブ切り替えを前提に整理する

実装前に大きなレイアウト変更を行う場合は、変更方針を説明すること。

---

# 16. 現時点の結論

NEON REQUIEMのUIでは、スクロールを完全に避ける必要はありません。

施設やアップグレードが増える以上、スクロール領域は必要です。

ただし、以下は常に守ります。

- 重要な数値と操作は上部ヘッダーに集約する
- メッセージは上部ヘッダー直下の横長バーで大きめに扱う
- 常に少なくとも1人の注目アイドルが画面に見える
- 全アイドルを無理に常時表示しない
- 施設・アップグレード・アイドル・記録は分類する
- 右側コンテンツはタブまたはカテゴリ切り替え式を前提にする
- 左下メニューは現時点では採用しない
- 設定は右上に1つだけ置く
- 美少女アイドルゲームとしての顔を失わない

この方針により、NEON REQUIEMは「実用的なインクリメンタルゲームUI」「世界観を語るメッセージ」「アイドルゲームとしてのキャラクター存在感」を両立することを目指します。
