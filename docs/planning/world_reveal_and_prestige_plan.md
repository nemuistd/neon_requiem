# 世界観開示と転生 / 再固定 方針

## 位置づけ

この文書は、世界観の開示を急がず、転生 / 再固定 / 再観測後にも新しい気付きが発生する構造を整理する。

## 基本原則

世界観の核心は序盤で解決しない。
プレイヤーが長期的に進行し、時には転生 / 再固定 / 再観測を経て、以前の出来事を再解釈できるようにする。

## 開示レイヤー

### Layer 1: 表層復興

- 忘れられた施設は廃れる。
- 人が集まらない街は寂れる。
- ライブで人が戻り、街が少し明るくなる。
- 「灯るさ」は区画の安定度として機能する。

### Layer 2: 異常の気配

- 名前を呼ぶと場所が安定する。
- 見られない道が欠ける。
- 録音した声に反響が返る。
- 忘却が比喩ではない気配が出る。

### Layer 3: 技術的断片

- 観測者数
- 認知固定率
- 霞曝露耐性
- アンカー個体
- 祈念負荷

ただし、これらは通常UI名にしない。

### Layer 4: 深層

- 祈念工学
- 聖歌
- 主人公の封印記憶
- 旧計画
- 外的現象としての霞

### Layer 5: 転生 / 再固定 後

- 1周目では意味が分からなかった序盤文言の再解釈。
- 記憶断片の持ち越し。
- 序盤記録への追加行。
- 主人公がなぜ記憶を封印したのかに関する断片。
- 霞に呼応しないための安全な復興手段として、アイドル活動が再評価される。

## 転生システム候補名

通常UI候補:

- 再固定
- 灯し直し
- 再観測

深層ログ候補:

- 再観測
- 記憶封印
- 局所再固定

最初から正式名称を確定しなくてよい。

## 転生で持ち越す候補

- 一部記録の既読
- 記憶断片
- アイドルbondの一部
- 一部アイテム効果
- オフライン上限
- 深層施設の解放ヒント

## Codex禁止事項

- 転生システムを事前確認なしに実装しない。
- 転生名称を勝手に正式決定しない。
- 主人公の正体を確定しない。
- 霞を人格ある敵として確定しない。
- 祈念工学の全貌を序盤で明かさない。
