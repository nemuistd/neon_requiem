# 世界観開示と廻方針

## 位置づけ

この文書は、世界観の開示を急がず、廻後にも新しい気付きが発生する構造を整理する。
後から承認された `docs/fiction/world_design.md` と `docs/fiction/meguri_after_design.md` を優先し、この文書はそれらを実装計画へ翻訳する補助として扱う。

## 基本原則

世界観の核心は序盤で解決しない。
プレイヤーが長期的に進行し、時には廻を経て、以前の出来事を再解釈できるようにする。

## 開示レイヤー

### Layer 1: 表層復興

- 忘れられた施設は廃れる。
- 人が集まらない街は寂れる。
- ライブで人が戻り、街が少し明るくなる。
- 「灯るさ」は区画の安定度として機能する。

### Layer 2: 異常の気配

- 名前を呼ぶと場所が安定する。
- 見られない道が欠ける。
- 録音した声に反響が返る。
- 忘却が比喩ではない気配が出る。

### Layer 3: 技術的断片

- 観測者数
- 認知固定率
- 霞曝露耐性
- アンカー個体
- 祈念負荷

ただし、これらは通常UI名にしない。

### Layer 4: 深層

- 祈念工学
- 聖歌
- 主人公の封印記憶
- 旧計画
- 外的現象としての霞

### Layer 5: 廻後

- 1周目では意味が分からなかった序盤文言の再解釈。
- 記憶断片の持ち越し。
- 序盤記録への追加行。
- 主人公がなぜ記憶を封印したのかに関する断片。
- 霞に呼応しないための安全な復興手段として、アイドル活動が再評価される。
- 廻後バフの購入によって、記録の追記が開く。
- Ch.7 / 廻1で、深層観測所と霞山 澪を通じて霞・観測側の深層入口を開く。
- Ch.8 / 廻1で、工学記録保管区、祈念工学実験跡地、七城 皐月を通じて記憶・記録・祈念工学の断片を扱う。
- Ch.9 / 廻2で、再観測拠点、名前のない劇場、白霧 燐、バインダー全文解読を扱う。ただし物語上の解決は提示しない。現在実装では Ch.9 収束補強まで最小実装済みで、主人公の正体、霞の意志、旧計画の全貌は確定していない。

## 廻システム名

正式名称:

- 廻（めぐり）

`docs/fiction/world_design.md` の承認済み正史に従い、転生 / 再固定 / 再観測系の正式システム名は「廻」とする。
「再固定」「再観測」「灯し直し」は、施設名・ログ内概念・演出語として残してよいが、正式なシステム名候補としては扱わない。

深層ログ候補:

- 再観測
- 記憶封印
- 局所再固定

最初から正式名称を確定しなくてよい。

## 廻で持ち越すもの

- 解放済み記録
- 既読記録
- 記録アノテーションの解放状態
- 記憶断片
- 購入済み廻後バフ
- アイドルbondの痕跡フラグ
- 深層施設の解放ヒント

アイドルbond数値、施設レベル、歌・アイテムの購入状態、灯るさ現在値は基本的にリセットする。記憶断片は、通常プレイ中の常時生産ではなく、廻実行時に全周回累計の灯るさ獲得量から清算する。

## Codex禁止事項

- 廻本体の大型仕様変更や、Ch.7以降の深層展開を事前確認なしに実装しない。
- 主人公の正体を確定しない。
- 霞を人格ある敵として確定しない。
- 祈念工学の全貌を序盤で明かさない。
- fiction で承認済みの Ch.7以降の章・施設・アイドル配置を、古い planning 側の未定候補へ戻さない。
