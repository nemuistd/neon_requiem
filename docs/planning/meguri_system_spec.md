# 廻本体 仕様メモ

## 位置づけ

この文書は、廻本体の基本仕様と今後の拡張判断を整理する。

正史・物語面は `docs/fiction/world_design.md` と `docs/fiction/meguri_after_design.md` を優先する。この文書は、それらを実装タスクへ落とすための仕様メモであり、v1 実装済みの内容と将来調整対象を併記する。

現在の実装には、再固定中枢到達、任意の廻実行、記憶断片清算、廻後バフ購入画面、初期4件の記録追記表示が入っている。Ch.7以降の新施設、新アイドル、白霧 燐、Ch.9収束は未実装。

## 基本方針

- 廻前のコンテンツは、現在の進行順に沿って一通り解放される。
- プレイヤーが廻前の最後の導線まで到達すると、自然に `再固定中枢` へ至る。
- 廻は強制ではなく、プレイヤーが任意に選ぶ。
- 廻の実行は大きな状態変更なので、実行前にリセット対象と引き継ぎ対象を明示する。
- 廻は「転生特典だけを受け取る操作」ではなく、「街を一度たたみ、痕跡を次へ持ち越す操作」として扱う。

## 発動条件

初回廻の発動条件は、廻前コンテンツを順番に進めた結果として満たされる形にする。

v1 実装済み条件:

- 地下通路修復区画 Lv5。
- `修復の仮歌` 取得。
- `再固定中枢` Lv1。

将来調整候補:

- 地下通路修復区画の終盤レベルに到達している。
- 廻前の主要記録・歌・アイテム・アイドル解放の導線が一通り完了している。

廻が実行可能になった後は、実行前の確認画面で「今ここで廻ると獲得できる記憶断片の見込み」を表示する。これは実際の付与処理と同じ算出式によるプレビューであり、廻を実行するまでは記憶断片を付与しない。

## リセット対象

廻を実行すると、通常進行で積み上げたものはほぼリセットする。

- 灯るさの現在値
- 施設レベル
- 歌の購入状態
- アイテムの購入状態
- アイドルの bond 数値
- 廻前の一時的な UI 状態

注目アイドルの選択は、保存実装上はリセットしても保持してもよい。ただし、選択中のアイドルが再解放前の場合は、表示上は未解放扱いに戻す。

## 引き継ぎ対象

プレイヤーの負担を減らし、世界に痕跡が残る感覚を出すため、記録まわりは強く保持する。

- 解放済み記録
- 既読記録
- 廻後アノテーションの解放状態
- 廻回数
- 記憶断片
- 購入済み廻後バフ
- アイドルの認識フラグ
- バフ購入に紐づく追記表示フラグ

アイドルの認識フラグは、bond 数値そのものではなく、「以前の廻で一定以上関わった」という痕跡として保持する。これにより、bond はリセットされても、薄明の記憶や微細なセリフ変化へ接続できる。

## 記憶断片

記憶断片は、廻を実行した時に獲得する引き継ぎ可能な資源とする。

基本方針:

- 記憶断片は通常プレイ中に常時生産しない。
- 記憶断片は、全周回における灯るさ獲得総量に応じて算出する。
- 廻実行時に、総量から算出される累計獲得可能量と、これまでに獲得済みの記憶断片累計との差分を付与する。
- これにより、同じ総量から何度も記憶断片を受け取ることを防ぐ。
- 記憶断片は廻をまたいで保持する。
- 消費済み記憶断片は戻さない。
- 廻前の確認画面では、同じ計算に基づく獲得見込みを表示する。
- 獲得見込みには、購入済みの記憶断片獲得量バフを反映する。
- まだ購入していない候補バフ、または廻後の清算画面でこれから購入するバフは、今回の獲得見込みには含めない。

実装時の注意:

- プレビュー表示と実際の付与処理は、同じ純粋関数を使う。
- 「購入済みバフを反映した獲得見込み」と「廻実行時に実際に付与された記憶断片」が一致するテストを入れる。
- UI 表示用に丸めた値と、保存・付与に使う値の丸め位置を揃える。
- 記憶断片獲得量バフは、清算前の基礎獲得量に適用するのか、差分付与量に適用するのかを実装前に固定する。

実装時には、以下のような保存値が必要になる可能性がある。

- 全周回累計の灯るさ獲得量
- 記憶断片の現在値
- 記憶断片の累計獲得量

具体的な算出式は未決定。灯るさの指数的な伸びをそのまま反映すると廻後バフが急激に膨らむため、対数、平方根、段階テーブルなど、伸びを圧縮する式を検討する。

## 廻後バフ

廻後バフは、記憶断片を消費して購入する。

購入タイミング:

- バフを購入できるのは、廻実行直後の清算画面、または廻後の専用バフ購入フェーズに限定する。
- 通常プレイ中は、購入済みバフと候補バフの確認だけ可能にする。
- 通常プレイ中に施設や灯るさを維持したままバフだけ購入できる状態にはしない。

画面方針:

- バフ購入は専用画面で行う。
- 既存の施設、歌、アイテム、記録、アイドル画面へ無理に詰め込まない。
- 通常プレイ中でも、購入済みバフの一覧と次回廻時に購入できそうな候補は確認できるようにする。
- 廻実行前の確認画面では、現在の進行で廻った場合に獲得できる記憶断片の見込みを表示する。
- この見込みは、購入済みの記憶断片獲得量バフを含めた最終見込みとして表示する。
- 廻実行直後は、獲得した記憶断片、購入可能バフ、購入済みバフをまとめて確認できるようにする。

候補カテゴリ:

- 灯るさ生産倍率
- ライブ獲得量
- オフライン報酬
- bond 上昇速度
- 記録アノテーション解放
- 記憶断片獲得効率

## アノテーション

廻後アノテーションは、バフ購入と連動して解放する。

- 記録の追記を直接コストで購入しない。
- バフを購入すると、そのテーマに関連する記録の追記が表示される。
- 解放済み記録に追記が追加された場合は、記録一覧で新しい追記があることを示す。
- 既読状態は維持するが、追記が増えた記録は再読する意味が分かる表示にする。

この方針により、「強化を買う」と「世界の読み方が変わる」が同時に起きる。

## 再固定中枢

`再固定中枢` は、実装上は通常施設に近い形で扱う。

役割:

- 廻前最後の導線として到達する施設。
- 廻選択肢を開く物語上の鍵。
- 灯るさを生産する通常施設。
- 記憶断片を常時生産する施設ではない。

世界観上の設定:

`再固定中枢` は、復興した街の状態を一時的に固定し、次の廻へ持ち越せる痕跡へ圧縮する場所である。施設が稼働している間に記憶断片を少しずつ作るのではなく、廻を選んだ瞬間に、これまで積み上げた灯るさ、記録、名前、観測の痕跡を記憶断片へ変換する。

この扱いにすると、再固定中枢はゲーム経済上は扱いやすい通常施設でありながら、物語上は廻の意味を支える中心施設になる。

## 実装分割案

1. 保存データ拡張
   - 廻回数、全周回累計灯るさ、記憶断片、廻後バフ、認識フラグを追加する。
2. 廻実行処理
   - リセット対象と引き継ぎ対象を確定し、保存マイグレーションを入れる。
3. 記憶断片清算
   - 廻実行時に記憶断片を付与する。
4. 廻後バフ画面
   - 専用画面で購入・確認を行う。
5. アノテーション表示
   - バフ購入に連動して記録追記を表示する。
6. 再固定中枢と導線
   - 廻前最後の導線から廻選択肢へつなげる。
