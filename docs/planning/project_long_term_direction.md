# NEON REQUIEM 長期方針

## 位置づけ

この文書は、ChatGPTとの会話で共有された長期方針をCodexに伝えるための参照文書である。

世界観の正史は `docs/worldbuilding.md` を優先する。
現在実装済み仕様は `docs/current_spec.md` を優先する。
この文書は、今後の拡張判断・実装順・Codexの判断補助として使う。

## 長期開発の前提

- アイドルは将来的に10人以上へ増加する。
- 施設も多数追加する。
- 現在実装済みのアイドルだけで物語を完結させない。
- 現在実装済みの施設だけで世界観を急いで解決しない。
- 世界観の核心開示は、廻後に段階的に行ってよい。
- 廻後バフや記憶断片がないと到達しにくい深層施設・深層記録で、新しい気付きが解放される構造を許容する。
- 最善の形を優先し、現時点の実装済みコンテンツだけで無理に物語を閉じない。

## 物語とゲーム体験の中心

暗い世界観は存在するが、ゲーム体験の中心は以下に置く。

- 回復
- 居場所づくり
- 復興
- 名前を呼び続けること
- 忘れられた場所をもう一度人の記憶に戻すこと
- アイドルを搾取するのではなく支えること

## 開示を急がない方針

この作品の世界観には、霞、祈念工学、聖歌、主人公の記憶封印、旧計画などの深層要素がある。

ただし、これらは序盤で解決しない。
序盤は表向きにはアイドル拠点運営・復興ゲームとして始める。

開示は以下の順でよい。

1. 忘れられた場所は廃れる、という一般的現象に見える。
2. 名前を呼ぶ、見られる、覚えられることが場所の安定に影響する気配が出る。
3. 忘却が比喩ではなく物理現象であることが見える。
4. 観測者数、認知固定率、アンカーなどの技術語が断片的に出る。
5. 祈念工学、聖歌、主人公の封印記憶が深層で出る。
6. 廻後に、序盤の出来事が別の意味を持って見える。

## Codexへの重要注意

Codexは以下を勝手に確定しないこと。

- 主人公が過去の本人なのか、代行存在なのか。
- 霞に人格や意志があるか。
- 旧計画の全貌。
- 聖歌の具体的な発動条件。
- 祈念工学の完全な技術体系。
- 世界崩壊の全原因。
- 廻の最終仕様。
- 10人以上のアイドル全員の最終役割。

必要な場合は「未確定」「将来検討」「深層開示で扱う」と書く。
