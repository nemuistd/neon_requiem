# 施設進行・地下礼拝堂配置方針

## 位置づけ

この文書は、施設数が将来的に増えることを前提に、施設の役割と開示順を整理する。
承認済みの章構造、施設名、解放条件の大枠は、後から作られた `docs/fiction/world_design.md` と `docs/fiction/content_design.md` を優先する。この文書の古い候補名や仮配置が fiction と食い違う場合は、fiction を正として更新する。

## 大前提

施設は今後多数追加される。
現在実装済みの施設だけで世界観やメインストーリーを完結させない。

特に地下礼拝堂は、聖歌、祈念工学、主人公の封印記憶などに接続しうる強い意味を持つ施設である。
そのため、序盤メイン導線に固定するかどうかは慎重に検討する。

## 推奨方針

地下礼拝堂は削除しない。
ただし、序盤2施設目や序盤必須メイン施設として扱いすぎない。

推奨位置づけ:

> 中盤以降に開く、最初の深層施設候補。
> ただし、地下礼拝堂だけで深層を解決しない。

現在実装上は、路地裏ステージ → ネオン掲示板 → 薄明通り案内所 → 仮設配信ブース → 記憶図書館 → 録音保管庫 → 古い放送室 → 地下広場 → 名前の記録壁 → 地下礼拝堂 → 地下通路修復区画 → 再固定中枢の順で進行する。
今後の多数施設化では、この順序を固定された最終形とは見なさない。
ネオン掲示板後に導線施設や記憶施設を挟み、地下礼拝堂を分岐施設または中盤以降の深層施設として扱う案を推奨する。

## 施設カテゴリ

### 1. 表層復興施設

序盤向け。
深層語を出さず、復興ゲームとして自然に見える施設。

目的:

- 基本ループを覚える。
- ライブ、告知、人の流れ、設備復旧を自然に見せる。
- 「灯るさ」が街の安定に関わることを、説明しすぎず体験させる。

候補:

- 路地裏ステージ
- ネオン掲示板
- 仮設配信ブース
- 小さな客席
- 修復工房
- ファン待機所

### 2. 導線施設

序盤後半から中盤手前。
人の流れ、道、告知、霞の薄い異常性を扱う。

目的:

- ネオン掲示板の後、地下礼拝堂へ直行しすぎないための緩衝地帯にする。
- 朝霧結や追加アイドルの役割を自然に作る。
- 道が消える、案内が必要になる、人が戻ると区画が安定する、といった異常の気配を扱う。

候補:

- 薄明通り案内所
- 霞避けの標識灯
- 消えかけた路線図
- 地下駅跡

### 3. 記憶施設

中盤向け。
名前、記録、録音、記憶、観測者数の断片を扱う。

目的:

- 「覚えられること」「名前を呼ばれること」が街の安定に関わる気配を強める。
- 記録タブや交流値、将来のファン資源と接続しやすくする。
- 深層語を出す場合も、まだ断片的に留める。

候補:

- 記憶図書館
- 録音保管庫
- 古い放送室
- 名前の記録壁

### 4. 深層施設

中盤後半から後半向け。
祈念工学、聖歌、主人公の記憶封印、旧計画へ接続する。

目的:

- 表層復興で見えていた現象の裏側を、断片として開示する。
- 深月詩乃や深層担当アイドルの導線を作る。
- 世界観の核心に近いが、全貌は確定しない。

候補:

- 地下礼拝堂
- 地下通路修復区画
- 深層観測所
- 工学記録保管区
- 祈念工学実験跡地

### 5. 廻起点・廻後施設

廻の起点、または廻後に意味が変わる施設。

目的:

- 1周目では意味が分からなかった施設や記録を読み替える。
- 序盤施設に追加の意味を与える。
- 深層施設のすべてを1周目で解決しない構造を作る。

候補:

- 再固定中枢（最小実装済み）
- 再観測拠点
- 名前のない劇場

再観測拠点と名前のない劇場は、fiction 正史では Ch.9 / 廻2の施設として扱う。現在実装では、Ch.9 起点の最小縦切りとして廻2以降に解放される。

## 地下礼拝堂の扱い

地下礼拝堂は、以下のいずれかとして扱う。

### 案A: 現状維持

路地裏ステージ → ネオン掲示板 → 地下礼拝堂。

メリット:
- 実装済みの進行を維持できる。
- 深月詩乃の導線が明確。

デメリット:
- 10人以上のアイドルと多数施設を前提にすると、深層へ進むのが早い。
- 礼拝堂が序盤メインラインに固定されやすい。

### 案B: 分岐施設化（採用中）

ネオン掲示板後に、薄明通り案内所や記憶施設を挟み、地下礼拝堂は分岐または中盤施設にする。

推奨。
理由:
- 礼拝堂の重みを保てる。
- 朝霧結や追加アイドルの導線を確保できる。
- 世界観開示を急がずに済む。

想定進行例:

1. 路地裏ステージ
2. ネオン掲示板
3. 薄明通り案内所
4. 消えかけた路線図、地下駅跡、仮設配信ブースなどの表層 / 導線施設
5. 記憶図書館、録音保管庫などの記憶施設
6. 地下礼拝堂を分岐施設または中盤以降の深層施設として解放

この案では、地下礼拝堂を削除せず、重みを保ったまま後ろへ置く。
深月詩乃の導線も、地下礼拝堂に直結させるだけでなく、記憶施設や録音保管庫を経由して段階的に近づけられる。

現行実装では、名前の記録壁 Lv3 を地下礼拝堂の解放条件とすることで、名前と記録の章を通ってから Chapter 5 に入る。

### 案C: 転生前後の深層施設化

地下礼拝堂をさらに後ろへ送る。

メリット:
- 深層開示の重みが最大になる。

デメリット:
- 既存コンテンツの再設計が大きい。

案Cは、廻後の大型展開やCh.7以降の設計が固まるまで採用を急がない。
地下礼拝堂を後ろへ送りすぎると、既存の深月詩乃や礼拝堂関連記録の扱いを大きく見直す必要がある。

## ネオン掲示板後の候補導線

ネオン掲示板は、告知と人の流れを作る施設である。
その後に置く施設は、地下礼拝堂へ直行するよりも、表向きの復興ゲームから異常の気配へ自然に橋をかける役割が望ましい。

候補:

| 候補施設 | カテゴリ | 役割 | 接続しやすい要素 |
| --- | --- | --- | --- |
| 薄明通り案内所 | 導線施設 | 道案内、人の流れ、迷いやすい通路 | 朝霧結、オフライン報酬、施設解放補助 |
| 消えかけた路線図 | 導線施設 | 地下都市の移動導線と欠ける道 | 地下駅跡、記録、霞の気配 |
| 仮設配信ブース | 表層復興施設 | 遠くの人へ公演を届ける | 配信ブース系アイドル、ライブ獲得、ファン資源 |
| 小さな客席 | 表層復興施設 | 場所に人が戻る感覚 | ライブ獲得、交流値、序盤記録 |
| 記憶図書館 | 記憶施設 | 名前や記録を整理する | 記録解放、観測者数の断片 |
| 録音保管庫 | 記憶施設 | 声や公演の保存 | 歌効果、記録、深月詩乃への導線（最小実装済み） |
| 古い放送室 | 記憶施設 / 表層復興施設 | 古い放送設備を修理する | 録音、記録、音が残る異常性（最小実装済み） |
| 地下広場 | 表層復興施設 / 記憶施設 | 人が集まり名前を呼ぶ場所 | 陽向小春、交流値、名前の効果（最小実装済み） |
| 名前の記録壁 | 記憶施設 | 呼ばれた名前や戻ってきた名前を見える場所に残す | 地下広場、名称固定、地下礼拝堂前の接続（最小実装済み） |
| 地下通路修復区画 | 深層施設 / 導線施設 | 倒壊した通路を復元し、再固定中枢の手前まで進める | 継ノ端 桜子、修復記録、廻前の停止点（最小実装済み） |
| 再固定中枢 | 廻起点施設 | 復興した街を一度固定し、廻へ進む判断を提示する | 記憶断片、廻後バフ、記録追記（最小実装済み） |

この表は廻前導線の設計候補であり、Ch.7以降の fiction 正史施設を未定化するものではない。

`再固定中枢` と廻本体の最小仕様は実装済み。以後の拡張では、`docs/fiction/content_design.md`、`docs/planning/meguri_system_spec.md`、`docs/current_spec.md` を確認し、Ch.7以降の新施設や新アイドルを別タスクとして扱う。

## 判断基準

新施設を追加する場合は、以下を確認する。

- 序盤に深層語を出しすぎていないか。
- その施設が、表層復興、導線、記憶、深層、廻起点・廻後のどこに属するか説明できるか。
- 既存施設と役割が重複しすぎていないか。
- 対応するアイドルや記録、アイテム、歌との接続が自然か。
- 地下礼拝堂を単なる早期施設に戻していないか。
- 世界観の核心を一施設だけで解決していないか。

## Codexへの指示

Codexは、地下礼拝堂を序盤メイン導線に戻さない。
地下礼拝堂に関わる変更を行う場合は、必ず理由と影響範囲を説明する。
新施設候補を扱う場合も、名前や解放条件を勝手に正式決定せず、必要に応じて設計候補として記述する。ただし、fiction で承認済みの Ch.7以降施設は未定候補へ戻さず、実装済み / 未実装の状態を分けて扱う。

章進行と次施設候補の詳細は、`chapter_progression_design.md` と `next_facility_candidates.md` を参照する。
