# NEON REQUIEM 実ブラウザQA方針と試行記録

## 現在の方針

- 通常の回帰確認は `npm.cmd run build`、`npm.cmd run check:content`、必要に応じて `npm.cmd run test` を優先する。
- ゲームロジック、保存データの変換、コンテンツ定義の検証は、ブラウザ操作よりも Node / Vitest / content check で決定的に確認する。
- UI 表示、ボタン操作、タブ切替、localStorage の保存復元、コンソールエラー、モバイル幅の崩れは、必要な範囲に絞って in-app Browser で確認する。
- 実ブラウザQAは毎回フルチェックにしない。変更内容に応じて、下記チェック項目から必要なものを選ぶ。
- in-app Browser が利用できない、または不安定な場合のみ、Chrome + Chrome DevTools Protocol などの代替手段へ切り替える。
- `node_repl` / `mcp__node_repl__js` は環境により利用可否が変わるため、固定的な前提にしない。使える場合は in-app Browser 操作の接続経路として利用してよいが、ロジック検証の標準経路にはしない。
- セッション制約で Vite dev server の常駐起動が難しい場合は、`npm.cmd run build` 後に `dist` を一時配信して実ブラウザ確認してよい。

## 推奨する確認順

1. `npm.cmd run build`
2. コンテンツ定義やデータ構造を触った場合は `npm.cmd run check:content`
3. ゲームロジック、保存、効果計算を触った場合は `npm.cmd run test`
4. UI や保存復元に影響がある場合は、必要範囲だけ in-app Browser で実ブラウザQA
5. in-app Browser が詰まる場合のみ、Chrome + CDP などへ切り替え

## 実ブラウザQAの選択項目

| 区分 | 項目 |
| --- | --- |
| 初期表示 | 新規セーブで起動し、`NEON REQUIEM`、灯るさ、毎秒、ライブボタン、注目アイドルが表示される |
| ライブ | `ライブする` で灯るさが増え、メッセージが更新される |
| 保存復元 | リロード後も灯るさ、施設Lv、注目アイドル、購入状態、既読状態が残る |
| 施設 | 灯るさ10で路地裏ステージを強化でき、Lvと次コストが更新される |
| タブ | 復興区画、歌、アイテム、アイドル、記録を開ける |
| 設定 | 設定画面を開閉できる。セーブ削除は必要な場合だけ実行する |
| 進行解放 | ネオン掲示板、地下礼拝堂、追加アイドル、歌、アイテム、記録の解放条件が表示・反映される |
| レスポンシブ | モバイル幅で主要表示と操作ボタンが大きく崩れない |
| コンソール | 実ブラウザ操作中にエラーが出ていない |

## 既知の注意点

- 実ブラウザQAは有効になったが、すべての確認をブラウザ操作に寄せない。ロジックやデータ検証は自動テストと content check を優先する。
- Vite dev server の常駐起動がセッション制約で失敗する場合がある。その場合は、`dist` の一時配信で UI 確認を代替してよい。
- 一時サーバー、ログ、スクリーンショット、trace は原則としてワークツリー外に置き、QA 後に不要なものを残さない。
