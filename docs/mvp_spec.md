# NEON REQUIEM MVP 仕様書

## 位置づけ

この文書は、当初の MVP v0.1 の目的と、現在の実装がどこまで拡張されたかを記録する文書である。

現行仕様の詳細は [current_spec.md](current_spec.md) を正とする。世界観・用語体系は [worldbuilding.md](worldbuilding.md) を正とする。手動確認手順は [manual_qa_checklist.md](manual_qa_checklist.md) を参照する。

## 当初MVPの目的

MVP の目的は、ブラウザ向けインクリメンタルゲームとしての最小ループを検証することだった。

確認したいこと:

- 手動生産が動く。
- 強化購入が動く。
- 自動生産が動く。
- 保存と読み込みが動く。
- オフライン進行が動く。
- 後からキャラや施設を増やしやすい構造になっている。

## 当初MVPの最小ループ

1. プレイヤーが「ライブする」ボタンを押す。
2. 灯るさが増える。
3. 灯るさを消費して「路地裏ステージ」を強化する。
4. 路地裏ステージが毎秒少しずつ灯るさを生産する。
5. ゲーム状態を localStorage に保存する。
6. ゲームから離れていた時間に応じて、簡易的なオフライン報酬を与える。

この最小ループは現在も維持している。

## 現在の進行順

現在の序盤から中盤への進行は以下。

1. 路地裏ステージから開始する。
2. 路地裏ステージ Lv3 で「古いネオン管」が解放される。
3. 路地裏ステージ Lv5 で「路地裏のイントロ」と「手書きの告知ポスター」が解放される。
4. 路地裏ステージ Lv10 で「ネオン掲示板」が解放される。
5. ネオン掲示板 Lv3 で「録音済みの短い挨拶」が解放される。
6. ネオン掲示板 Lv5 で「朝霧 結」が解放される。
7. ネオン掲示板後は、薄明通り案内所、仮設配信ブース、記憶図書館、録音保管庫、古い放送室、地下広場、名前の記録壁へ進む。
8. 名前の記録壁 Lv3 で「地下礼拝堂」が解放される。
9. 地下礼拝堂 Lv2 で「礼拝堂のハーモニー」が解放される。
10. 地下礼拝堂 Lv3 で「深月 詩乃」と技術寄りの記録が解放される。
11. 地下礼拝堂 Lv8 で「薄明のコーラス」が解放される。
12. 地下礼拝堂 Lv8 と「礼拝堂のハーモニー」取得で「地下通路修復区画」が解放される。
13. 地下通路修復区画 Lv1 で「継ノ端 桜子」が解放される。
14. 地下通路修復区画 Lv5 と「修復の仮歌」取得で「再固定中枢」が解放され、廻を実行できる。
15. 初回廻後、再固定中枢 Lv3 から「深層観測所」と「霞山 澪」へ進む。
16. 初回廻後、深層観測所 Lv5 から「工学記録保管区」「祈念工学実験跡地」「七城 皐月」へ進む。
17. 廻2以降、祈念工学実験跡地 Lv3 から「再観測拠点」「名前のない劇場」「白霧 燐」へ進む。
18. 白霧 燐 合流済み、名前のない劇場 Lv2 以降で「最後の名前」を取得し、バインダー封書・全文、残響公演、オープンエンド導線へ進む。

地下礼拝堂は序盤2施設目ではなく、中盤以降の深層施設として扱う。
再固定中枢以降の深層施設は、現行では最小縦切りとして実装済みである。詳細は [current_spec.md](current_spec.md) を参照する。

## 用語方針

- 数値リソース名は「灯るさ」。
- 演出文としての「灯り」は残してよい。
- 「聖歌」は通常システム名にしない。
- 「灯し手」は表向き呼称。
- 「アンカー」は旧工学側の管理的呼称。
- 序盤UIでは祈念工学、アンカー、祈念負荷、聖歌などの深層語を前面に出しすぎない。

## 技術目標

- 静的ブラウザアプリ。
- Vite + TypeScript。
- バックエンドなし。
- ログインなし。
- データベースなし。
- 保存は localStorage。
- npm script でローカル起動できる。
- GitHub Pages へ載せやすい構成を維持する。

## 現在もスコープ外のもの

- 複数リソース
- ガチャ
- 課金
- 音声
- 複雑なストーリーシステム
- ログイン
- バックエンド
- クラウドセーブ
- 詳細なセーブ復旧UI
- 主人公の正体、霞の意志、旧計画の全貌を確定する演出

## historical: 当初MVPとの差分について

過去のMVP案では、基本リソースを「灯り」と呼び、2施設目として地下礼拝堂を早期に出す想定があった。

現在は世界観方針に合わせ、数値リソースを「灯るさ」とし、地下礼拝堂は名前の記録壁の後に出る深層寄りの施設へ再配置している。

古い記述は設計経緯として参照してよいが、現行仕様としては扱わない。
