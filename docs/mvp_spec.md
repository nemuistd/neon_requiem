# NEON REQUIEM MVP 仕様書

## 現在の状態

この文書は、当初の MVP v0.1 の仕様を記録する文書である。

現在の実装は、MVP v0.1 を土台に Phase 2 の最小拡張が入っている。

現在追加済みの主な差分:

- 施設は `facilities` 形式で管理する。
- 施設は「路地裏ステージ」と「地下礼拝堂」の2つ。
- アイドルは「音羽 灯里」「深月 詩乃」「朝霧 結」の3人。
- 深月 詩乃と地下礼拝堂は、路地裏ステージ Lv 10 で解放される。
- 地下礼拝堂は全体倍率ではなく、灯りを直接生産する施設として扱う。
- 歌タブと、1回購入型の歌アップグレードを実装済み。
- 記録タブと、進行条件で解放される読み物を実装済み。
- セーブデータは `saveVersion: 5` を使う。
- UI は上部操作パネル、メッセージバー、注目アイドル枠、復興区画パネルに再配置済み。
- 開発用に `window.__NEON_DEBUG__` を用意している。

当初MVPの最小ループ自体は維持している。

## 目的

MVP の目的は、インクリメンタルゲームとしての最小ループを検証することである。

確認したいこと:

- 手動生産が動く。
- 強化購入が動く。
- 自動生産が動く。
- 保存と読み込みが動く。
- オフライン進行が動く。
- 後からキャラや施設を増やしやすい構造になっている。

MVP は完成版ゲームではない。

## 技術目標

- 静的ブラウザアプリ。
- Vite + TypeScript を推奨。
- バックエンドなし。
- ログインなし。
- データベースなし。
- 保存は localStorage で十分。
- 標準的な npm script でローカル起動できる。
- 将来的に GitHub Pages へ載せやすい構成にする。

## MVP ゲーム内容

### リソース

MVP のリソースは 1 つだけ。

- 灯り

灯りは以下で増える。

1. 「ライブする」ボタンを押す。
2. 路地裏ステージによる自動生産。
3. 購入済みの歌による効果。
4. ゲームを離れていた時間に応じたオフライン報酬。

### 手動アクション

ボタン:

- 表示: ライブする
- 効果: クリックごとに 灯り +1

現在の実装では、歌「路地裏のイントロ」取得後はクリックごとに 灯り +2 になる。

### 初期アイドル表示

UI には、初期アイドル **音羽 灯里（おとは あかり）** を表示する。

表示する情報:

- 名前: 音羽 灯里
- 肩書き: 路地裏の歌姫
- 短いフレーバーテキスト

現在の実装では、アイドル画像を表示する。

現在の実装では、左側に注目アイドル枠を置き、音羽 灯里を初期表示する。

深月 詩乃は解放条件達成後に注目アイドルへ切り替えられる。

### 施設

当初 MVP の施設は 1 つだけ。

- 路地裏ステージ

持つ情報:

- レベル
- 現在の強化コスト
- 秒間灯り生産量

初期値:

- level: 0
- baseCost: 10 灯り
- costMultiplier: 1.15
- productionPerLevel: 0.1 灯り / 秒

コスト式:

```text
cost = floor(baseCost * costMultiplier ^ level)
```

生産量式:

```text
lightsPerSecond = alleyStageLevel * 0.1
```

現在の実装では、施設状態は `facilities` に拡張されている。

```text
facilities.alleyStage.level
facilities.undergroundChapel.level
```

地下礼拝堂:

- 解放条件: 路地裏ステージ Lv 10
- baseCost: 100 灯り
- costMultiplier: 1.2
- productionPerLevel: 0.5 灯り / 秒

現在の秒間灯り量:

```text
lightsPerSecond =
  alleyStage.level * 0.1
  + unlocked(undergroundChapel) ? undergroundChapel.level * 0.5 : 0
```

強化挙動:

- 灯りが現在コスト以上なら強化できる。
- 強化するとコスト分の灯りを消費する。
- 路地裏ステージのレベルが 1 上がる。
- 秒間灯り生産量が即時に増える。

### 自動生産

時間経過に応じて灯りを増やす。

実装方針:

- タイムスタンプから差分時間を計算する。
- `lightsPerSecond * elapsedSeconds` を加算する。
- 表示上は読みやすく丸める。

### セーブデータ

localStorage を使う。

当初 MVP で最低限保存するもの:

- 灯り
- 路地裏ステージのレベル
- 最終保存時刻
- 可能ならセーブデータバージョン

現在の保存内容:

- `saveVersion: 5`
- `lights`
- `facilities.alleyStage.level`
- `facilities.undergroundChapel.level`
- `songs.*.purchased`
- `records.*.read`
- `lastSavedAt`

保存キー:

```text
neon-requiem-save-v1
```

保存タイミング:

- 強化など意味のある操作後。
- 数秒ごとの定期保存。
- 可能ならページ離脱前。

現在の実装では、保存前に現在時刻までの自動生産を反映する。

オートセーブはメインメッセージを上書きしない。

### 読み込み挙動

ページ読み込み時:

1. localStorage からセーブデータを読む。
2. セーブがなければ初期状態を作る。
3. 最終保存時刻から現在時刻までの差分を計算する。
4. 現在の自動生産量に基づいてオフライン報酬を計算する。
5. 灯りに加算する。
6. UI に反映する。

### オフライン報酬

MVP では単純な計算でよい。

```text
offlineLights = lightsPerSecond * offlineSeconds
```

報酬上限は後で考える。MVP では必須ではない。

読み込み時に、可能なら小さなメッセージを表示する。

```text
離れている間に、灯里が X 灯りを集めました。
```

## MVP UI 要件

画面に含めるもの:

- ゲームタイトル: NEON REQUIEM
- 短いサブタイトル
- 灯り量
- 秒間灯り量
- ライブするボタン
- 音羽 灯里のアイドルカード
- 路地裏ステージの強化カード
- オートセーブやオフライン報酬の簡単な表示

推奨レイアウト:

```text
[タイトル / サブタイトル]
[リソース表示]
[ライブするボタン]
[音羽 灯里のカード]
[路地裏ステージのカード]
[簡単なログ / オフライン報酬メッセージ]
```

現在のUI構成:

```text
[上部操作パネル: タイトル / 灯り / 毎秒 / ライブする / 設定]
[横長メッセージバー]
[メイン領域]
  左: 注目アイドル + アイドル切替
  右: 復興区画 / 歌 / 記録 のタブ切替
```

## MVP ビジュアル方向

MVP ではシンプルな CSS のみでよい。

目指す印象:

- 暗い背景
- ネオンの差し色
- 柔らかい発光
- 地下ステージ感
- 読みやすい文字

ただし、コアゲームループが動く前に見た目へ時間をかけすぎない。

## 明示的にスコープ外

MVP では実装しない。

- 複数リソース
- 転生システム
- ストーリーチャプター
- ガチャ
- ショップ
- 課金
- 音声
- 画像
- 高度なアニメーション
- ログイン
- バックエンド
- クラウドセーブ
- 設定画面

現在は Phase 2 として、複数アイドル、複数施設、歌アップグレード、記録の最小対応を実装済み。

ただし以下はまだスコープ外:

- 複雑なアイドル編成
- アイドル詳細画面
- アイテムシステム

## 手動確認チェックリスト

実装後に確認すること:

- アプリがローカルで起動する。
- 新規セーブで灯りが 0 から始まる。
- 「ライブする」を押すと灯りが 1 増える。
- 灯り 10 で路地裏ステージを強化できる。
- 強化時に灯りが消費される。
- 路地裏ステージのレベルが上がる。
- 強化後、秒間灯り量が増える。
- クリックしなくても時間経過で灯りが増える。
- ページ再読み込み後も灯りとステージ Lv が残る。
- 離れていた時間に応じてオフライン灯りが加算される。
- スコープ外の機能が勝手に追加されていない。

Phase 2 追加確認:

- 初期状態で深月 詩乃と地下礼拝堂が未解放表示になる。
- 未解放アイドルに解放条件が表示される。
- 路地裏ステージ Lv 10 で深月 詩乃と地下礼拝堂が解放される。
- 深月 詩乃を注目アイドルに切り替えられる。
- 歌タブを開ける。
- 路地裏ステージ Lv 5 で「路地裏のイントロ」が解放される。
- 「路地裏のイントロ」取得後、ライブ1回の獲得量が 2 灯りになる。
- 地下礼拝堂 Lv 1 で「礼拝堂のハーモニー」が解放される。
- 地下礼拝堂 Lv 5 で「薄明のコーラス」と朝霧 結が解放される。
- 記録タブを開ける。
- 初期解放記録が読める。
- 施設Lvや歌取得に応じて記録が解放される。
- 記録を読むと既読状態が保存される。
- 記録を読んでも灯りや生産量は変化しない。
- `window.__NEON_DEBUG__.resetSave()` で初期状態テストができる。
