# NEON REQUIEM MVP 仕様書

## 目的

MVP の目的は、インクリメンタルゲームとしての最小ループを検証することである。

確認したいこと:

- 手動生産が動く。
- 強化購入が動く。
- 自動生産が動く。
- 保存と読み込みが動く。
- オフライン進行が動く。
- 後からキャラや施設を増やしやすい構造になっている。

MVP は完成版ゲームではない。

## 技術目標

- 静的ブラウザアプリ。
- Vite + TypeScript を推奨。
- バックエンドなし。
- ログインなし。
- データベースなし。
- 保存は localStorage で十分。
- 標準的な npm script でローカル起動できる。
- 将来的に GitHub Pages へ載せやすい構成にする。

## MVP ゲーム内容

### リソース

MVP のリソースは 1 つだけ。

- 拍手

拍手は以下で増える。

1. 「ライブする」ボタンを押す。
2. 路地裏ステージによる自動生産。
3. ゲームを離れていた時間に応じたオフライン報酬。

### 手動アクション

ボタン:

- 表示: ライブする
- 効果: クリックごとに 拍手 +1

### 初期アイドル表示

UI には、初期アイドル **音羽 灯里（おとは あかり）** を表示する。

表示する情報:

- 名前: 音羽 灯里
- 肩書き: 路地裏の歌姫
- 短いフレーバーテキスト

MVP では画像は不要。装飾されたカード表示でよい。

### 施設

MVP の施設は 1 つだけ。

- 路地裏ステージ

持つ情報:

- レベル
- 現在の強化コスト
- 秒間拍手生産量

初期値:

- level: 0
- baseCost: 10 拍手
- costMultiplier: 1.15
- productionPerLevel: 0.1 拍手 / 秒

コスト式:

```text
cost = floor(baseCost * costMultiplier ^ level)
```

生産量式:

```text
applausePerSecond = alleyStageLevel * 0.1
```

強化挙動:

- 拍手が現在コスト以上なら強化できる。
- 強化するとコスト分の拍手を消費する。
- 路地裏ステージのレベルが 1 上がる。
- 秒間拍手生産量が即時に増える。

### 自動生産

時間経過に応じて拍手を増やす。

実装方針:

- タイムスタンプから差分時間を計算する。
- `applausePerSecond * elapsedSeconds` を加算する。
- 表示上は読みやすく丸める。

### セーブデータ

localStorage を使う。

最低限保存するもの:

- 拍手
- 路地裏ステージのレベル
- 最終保存時刻
- 可能ならセーブデータバージョン

保存キー:

```text
neon-requiem-save-v1
```

保存タイミング:

- 強化など意味のある操作後。
- 数秒ごとの定期保存。
- 可能ならページ離脱前。

### 読み込み挙動

ページ読み込み時:

1. localStorage からセーブデータを読む。
2. セーブがなければ初期状態を作る。
3. 最終保存時刻から現在時刻までの差分を計算する。
4. 現在の自動生産量に基づいてオフライン報酬を計算する。
5. 拍手に加算する。
6. UI に反映する。

### オフライン報酬

MVP では単純な計算でよい。

```text
offlineApplause = applausePerSecond * offlineSeconds
```

報酬上限は後で考える。MVP では必須ではない。

読み込み時に、可能なら小さなメッセージを表示する。

```text
離れている間に、灯里が X 拍手を集めました。
```

## MVP UI 要件

画面に含めるもの:

- ゲームタイトル: NEON REQUIEM
- 短いサブタイトル
- 拍手量
- 秒間拍手量
- ライブするボタン
- 音羽 灯里のアイドルカード
- 路地裏ステージの強化カード
- オートセーブやオフライン報酬の簡単な表示

推奨レイアウト:

```text
[タイトル / サブタイトル]
[リソース表示]
[ライブするボタン]
[音羽 灯里のカード]
[路地裏ステージのカード]
[簡単なログ / オフライン報酬メッセージ]
```

## MVP ビジュアル方向

MVP ではシンプルな CSS のみでよい。

目指す印象:

- 暗い背景
- ネオンの差し色
- 柔らかい発光
- 地下ステージ感
- 読みやすい文字

ただし、コアゲームループが動く前に見た目へ時間をかけすぎない。

## 明示的にスコープ外

MVP では実装しない。

- 複数アイドル
- 複数施設
- 複数リソース
- 転生システム
- ストーリーチャプター
- ガチャ
- ショップ
- 課金
- 音声
- 画像
- 高度なアニメーション
- ログイン
- バックエンド
- クラウドセーブ
- 設定画面

## 手動確認チェックリスト

実装後に確認すること:

- アプリがローカルで起動する。
- 新規セーブで拍手が 0 から始まる。
- 「ライブする」を押すと拍手が 1 増える。
- 拍手 10 で路地裏ステージを強化できる。
- 強化時に拍手が消費される。
- 路地裏ステージのレベルが上がる。
- 強化後、秒間拍手量が増える。
- クリックしなくても時間経過で拍手が増える。
- ページ再読み込み後も拍手とステージ Lv が残る。
- 離れていた時間に応じてオフライン拍手が加算される。
- スコープ外の機能が勝手に追加されていない。
