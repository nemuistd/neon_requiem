# Milestone Roadmap

## 位置づけ

この文書は、[Design Translation](20_design_translation.md) を段階的に満たすべき状態へ落としたロードマップである。

具体的な実装候補は [Task Backlog](40_task_backlog.md) で扱う。完了済みの作業分担や古い実行計画は `docs/product/archive/` に置く。

## 現在の到達点

2026-05-25 時点では、次の基盤が実装済みである。

- 初回3分導線、基本ループ、保存、オフライン報酬。
- 施設、歌、アイテム、アイドル、記録のタブ構造。
- `bond` を UI 上の「交流」として扱う基盤。
- アイドルの加入操作と、加入済みアイドルだけが効果を発動する仕組み。
- `passiveEffects` 複数効果モデルと、`focusEffects` の保留方針。
- 地下通路修復区画、修復の仮歌、再固定中枢。
- 廻本体、記憶断片、次断片ゲージ、清算画面、廻後バフ、記録追記通知。

現在仕様の詳細は `docs/current_spec.md` を優先する。

## 完了済みマイルストーン

### M0: 文書階層を整える

達成状態:

- `docs/product/` に、Product Charter から Backlog までの階層がある。
- `docs/product/archive/` と `docs/planning/archive/` に履歴文書を分離した。
- 実装済み仕様は `docs/current_spec.md`、長期構想は `docs/planning/` を優先する運用になっている。

### M1: 初回3分体験を固める

達成状態:

- 新規セーブでライブ、灯るさ、施設強化の関係を確認できる。
- `docs/manual_qa_checklist.md` に初回3分導線がある。
- 序盤 UI は深層語に依存せず進められる。

### M2: 成長と復興感を結びつける

達成状態:

- 初期メッセージ、ライブ、施設強化、歌、アイテム、オフライン報酬の文言を最小改善済み。
- 施設・歌・アイテム・記録が、数値効率だけでなく復興の手触りを補う方向に整理されている。

### M3: アイドルを個人として感じられる導線を作る

達成状態:

- アイドルは条件達成後に「声をかける」ことで加入する。
- 未加入アイドルの効果は発動せず、加入後に効果が加わる。
- 既存セーブの解放済みアイドルは加入済みに移行する。
- 交流イベントの最小仕様は `docs/planning/idol_roster_expansion_design.md` に整理済み。

### M4: 深層開示と長期進行を整える

達成状態:

- 記録の `surface`, `uncanny`, `technical`, `deep` の出し分けを点検済み。
- 廻後バフによる記録追記と、追記中心の通知ポリシーが実装済み。
- 主人公の正体、霞の意志、旧計画の全貌は未確定のまま保たれている。

### M5: タスクバックログを運用する

達成状態:

- 新規コンテンツ追加時の受け入れ条件テンプレートを追加済み。
- 完了済みタスクの詳細は現役バックログから分離し、履歴扱いへ移した。

### M6: 廻本体の土台を実装する

達成状態:

- 再固定中枢から廻を実行できる。
- 記憶断片の獲得見込み、次断片ゲージ、清算画面、廻後バフ購入がある。
- 廻で通常進行をリセットし、記録、記憶断片、廻後バフ、廻回数、以前の廻の痕跡を引き継ぐ。

## 次の候補マイルストーン

### M7: 廻後の再進行を厚くする

目的:

- 初回廻後、もう一度進める意味と見え方を増やす。

候補:

- 廻後の記録追記を増やす。
- 廻回数や購入済みバフに応じた、軽い再読・再進行表示を追加する。
- ただし Ch.7 以降の大型展開、白霧 燐、Ch.9 収束は別タスク化する。

### M8: 交流イベントの最小実装

目的:

- `bond` と `eventIdsRead` を、数値だけでなく個別の短い交流へ接続する。

候補:

- 初期実装済みアイドルの短い交流イベント。
- イベント既読管理。
- 記録タブとは別の、アイドル個別体験としての表示。

### M9: Ch.7 以降の新施設・新アイドル設計

目的:

- 廻後コンテンツの大型展開を、現在仕様と正史に合わせてタスク化する。

候補:

- Ch.7 以降の施設候補。
- 新アイドル候補。
- 白霧 燐の扱い。
- Ch.9 収束の前提整理。

## 未解決論点

- 廻後の再進行を、軽い数値加速中心にするか、記録追記中心にするか。
- 交流イベントをどの UI 面に置くか。
- Ch.7 以降の施設・アイドルをどの順で出すか。
- 実ブラウザQAを、どの粒度で毎タスクに含めるか。
