## 凡例

### 効果タイプ一覧

| タイプ識別子 | 意味 | 実装状況 |
|---|---|---|
| `facility.production.multiplier` | 全施設の灯るさ/秒生産に掛け算 | 実装済み |
| `manual.gain.add` | ライブ1回の獲得灯るさに加算 | 実装済み |
| `offline.reward.multiplier` | 基礎50%のオフライン報酬に掛け算 | 実装済み |
| `manual.gain.add.production.ratio` | ライブ1回の獲得灯るさに「全施設合計の灯るさ/秒 × 定数N」を加算 | **要実装** |
| `bond.rate.multiplier` | bond増加量に掛け算（全アイドル） | **要実装** |
| `record.unlock.cost.multiplier` | 記録解放コストに掛け算（1.0未満で軽減） | **要実装** |
| `item.cost.multiplier` | アイテム購入コストに掛け算（1.0未満で軽減） | **要実装** |
| `song.cost.multiplier` | 歌取得コストに掛け算（1.0未満で軽減） | **要実装** |
| `facility.production.multiplier.tag:deep` | deep施設に限定した施設生産倍率 | **要実装** |
| `memory.fragment.production.add` | 記憶断片リソース（廻後）の生産量加算 | **要実装（廻後）** |
| `rebirth.bonus.multiplier` | 廻後の全体ボーナス倍率に掛け算 | **要実装（廻後）** |

`offline.reward.multiplier` は、同じ時間のオンライン自然生産を基準にした基礎効率 `0.5` に乗算する。例: `x1.10` の場合、実効オフライン効率は `0.5 * 1.10 = 0.55`。今後の候補を実装に移すときは、複数バフを積んだ結果が意図せずオンライン自然生産を超えないか確認する。

### 解放条件タイプ

| タイプ識別子 | 意味 |
|---|---|
| `facility.level` | 指定施設が指定レベルに達している |
| `idol.bond` | 指定アイドルのbondが指定値以上 |
| `song.purchased` | 指定の歌を取得済み |
| `item.purchased` | 指定のアイテムを購入済み |
| `chapter` | 指定章に到達済み（廻フラグ連動） |
| `rebirth.count` | 廻の回数が指定値以上（廻後コンテンツ用） |

## 5. 実装メモまとめ

### 新規実装が必要な効果タイプ

| 効果タイプ | 用途 | 優先度 |
|---|---|---|
| `manual.gain.add.production.ratio` | ライブ獲得量への生産スケール加算（遠子パッシブ、広場のアンセム） | 高 |
| `bond.rate.multiplier` | bond増加量の乗算（巡パッシブ、書名ラベル） | 高 |
| `item.cost.multiplier` | アイテムコスト軽減（小春パッシブ） | 中 |
| `song.cost.multiplier` | 歌コスト軽減（皐月パッシブ、工学文書断片） | 中 |
| `facility.production.multiplier.tag:deep` | deepタグ施設限定倍率（澪パッシブ、深層の静謐等） | 高（Ch.7〜8で必要） |
| `memory.fragment.production.add` | 記憶断片リソース生産加算（燐パッシブ等）| 廻実装時 |
| `rebirth.bonus.multiplier` | 廻後の全体ボーナス乗算（燐パッシブ） | 廻実装時 |

### 施設タグの定義

| タグ | 対象施設ID |
|---|---|
| `infra` | `twilightPathGuide`, `undergroundPassageRepair`, `restabilizationCore`, `reobservationBase` |
| `deep` | `deepLayerObservatory`, `engineeringArchive`, `prayerEngineeringRuins` |

### 廻（転生）システム連動コンテンツ

廻システム実装時に合わせて有効化が必要なもの：
- 白霧 燐（全パッシブ・注目効果）
- 再観測拠点・名前のない劇場（廻フラグ解放条件）
- 最後の名前（廻後価格・効果変化）
- 灯し直しの歌（廻後効果強化）
- 再固定の手順書の写し（フレーバー・廻連動）
- 記憶断片リソースUI（再固定中枢 Lv1達成時に初表示）

### 注目時効果（focusEffects）設計方針

注目アイドルの選択がゲーム進行上の有利不利に影響しないよう、注目時効果（focusEffects）は採用しない。  
アイドルのパッシブ効果は常時適用（解放済みアイドル全員分）であり、誰を注目アイドルに設定しても損にならない設計を維持する。  
bondはそのアイドルの交流メモ（ストーリー記録）の解放にのみ影響し、数値的なゲームプレイ効果は持たない。  
`focusEffects` フィールドは型定義に存在するが、新規アイドルへの使用は行わない。
