# 17 次ゲーム内容拡張の設計タスク

## 目的

次に実装するゲーム内容拡張を設計する。
このタスクでは原則として実装しない。

## 候補

次の大きな候補は以下。

1. アイドル交流MVP
2. ファン資源MVP
3. 記録/章進行MVP
4. 薄明通り系の新施設・朝霧結の前倒し整理

## 推奨方針

次に実装するなら、まず「アイドル交流MVP」を推奨する。

理由:
- 注目アイドル機能に意味が出る。
- アイドルが単なる全体パッシブ装置になりにくい。
- 世界観上の「名前を呼ばれる」「顔を覚えられる」「個性を保つ」と相性がよい。
- まだ複数資源を増やさずに実装できる。

## 設計内容

`docs/next_gameplay_phase.md` を作成し、以下をまとめる。

### アイドル交流MVP案

- 注目アイドルにした状態でライブすると、対象アイドルの `bond` または `relationship` が増える。
- 最初は数値だけでよい。
- 一定値で短い交流記録が解放される。
- 交流記録は通常の記録とは分けてもよいし、記録カテゴリに「交流メモ」を追加してもよい。
- 「個性」はまだ本格資源にしなくてよい。将来拡張として設計だけ残す。

### State案

```ts
idols: Record<IdolId, {
  bond: number;
  eventIdsRead: string[];
}>
```

ただし今の `GameState` に `idols` stateがない場合、導入範囲とセーブ影響を明記する。

### Unlock案

- 音羽灯里 bond 5: 交流メモ1
- 音羽灯里 bond 20: 交流メモ2
- 深月詩乃 bond 5: 礼拝堂関連の短いメモ
- 朝霧結 bond 5: 薄明通り関連の短いメモ

### UI案

- 注目アイドル枠に交流値を表示
- アイドルタブにも交流値を表示
- 詳細ボタンはまだ無効でもよいが、将来の導線として残す

## 注意

- このタスクでは実装しない。
- 世界観ファイルにない重大設定を勝手に確定しない。
- 交流は支援的で、搾取的に見えない表現にする。

## 完了条件

- `docs/next_gameplay_phase.md` が追加される。
- 次に実装すべきPR分割案が含まれる。
- `npm run build`
- `npm run check:content`
