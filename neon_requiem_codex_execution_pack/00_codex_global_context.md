# Codex向け共通コンテキスト

あなたは `nemuistd/neon_requiem` の実装担当です。
このリポジトリは、地下都市・ネオン・歌・アイドル・復興をモチーフにした、Vite + TypeScript の静的ブラウザ向けインクリメンタルゲームです。

## 世界観の正史

`docs/worldbuilding.md` を正史としてください。
ユーザーが添付した `世界観設定.txt` も同等の基準として扱ってください。
未記載の世界観要素は、まだ固まっていないものとして扱い、勝手に重要設定を確定しないでください。

重要方針:

- 世界は人々の認知の平均に適応する。
- 多くの人に見られ、覚えられ、名前を呼ばれるものは安定する。
- 忘れられ、見捨てられ、意味を失ったものは実存の強度を失う。
- この現象は序盤では普通の社会的衰退に見えるが、実際には物理現象である。
- 外的現象の名前は「霞」。
- 旧技術体系は「祈念工学」。
- アイドル適性者の表向き名称は「灯し手」。
- 旧工学・管理側の呼称は「アンカー」。
- 「聖歌」は通常システム名ではなく、危険な深層用語。
- ライブは儀式ではない。応援は信仰ではない。歌は聖歌ではない。
- ゲーム体験の中心は搾取や絶望ではなく、回復、居場所づくり、復興。

## 技術方針

- 現状の軽量構成を維持する。
- React/Vue/Svelte等は導入しない。
- TypeScript strict を維持する。
- 依存追加は原則しない。
- コンテンツ定義はTypeScript定義のまま、ID・順序・状態生成・検証を自動化する。
- 1タスクで過大な変更をしない。
- 変更後は必ず `npm run build` と `npm run check:content` を実行する。
- 失敗した場合は、失敗内容と原因を明記し、可能なら修正する。

## セーブ方針

現段階ではリリース前でユーザーがいないため、旧セーブ互換性よりも最新版セーブデータの設計を優先してください。
ただし、開発中の確認のため、設定画面からのセーブ削除と `window.__NEON_DEBUG__` は維持してください。

## 実装単位

この計画は、以下の順で進めます。

1. 世界観ドキュメントとREADMEの整合
2. コンテンツ登録ヘルパー導入
3. `lights` から `resources.tomorusa` への移行
4. Requirement / Effect 共通化
5. 序盤施設進行の再配置
6. アイテムタブMVP
7. 記録・章・開示レベル整理
8. UI分割

各タスクでは、依頼された範囲を超えた先回り実装を避けてください。
