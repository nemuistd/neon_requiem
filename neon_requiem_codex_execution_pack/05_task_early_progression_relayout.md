# Codexタスク05: 序盤施設進行の再配置

## 目的

世界観方針に合わせ、地下礼拝堂を序盤2施設目から外し、より深い進行段階へ再配置してください。

## 背景

世界観正史では、「聖歌」は通常システム名ではなく危険な深層用語です。
地下礼拝堂は聖歌・祈念工学・深層ログに近いため、序盤に出すと世界観開示が早すぎます。

## 作業内容

1. 序盤2施設目として、以下のどちらかを追加してください。

第一候補:

```ts
id: "neonBoard"
name: "ネオン掲示板"
```

説明方針:

- 表向き: ライブ告知やアイドル名を掲示する施設。
- 裏側: 名前を見える場所に残し、街の認知固定を助ける。
- 序盤説明では深層語を出しすぎない。

第二候補:

```ts
id: "temporaryStreamingBooth"
name: "仮設配信ブース"
```

説明方針:

- 表向き: 離れた人にもライブを届ける仮設設備。
- 裏側: 観測者数を増やす伏線。

2. 地下礼拝堂は削除せず、表示順を後ろへ移動してください。
3. 地下礼拝堂の解放条件を、序盤施設の後に変更してください。
4. 深月詩乃の解放条件も、地下礼拝堂に合わせて後ろへ移動してください。
5. 朝霧結は「薄明通り」系の施設またはネオン掲示板/配信ブースの後に出せるよう、必要なら解放条件を調整してください。
6. 関連記録の解放条件を、進行順に合わせて調整してください。
7. READMEの主な数式と手動確認チェックリストを更新してください。

## 注意点

- 地下礼拝堂を消さない。
- 「聖歌」を序盤UIの通常語にしない。
- 新施設の数式は強すぎないようにする。
- 既存のアイドル画像や基本説明は、必要最小限の変更に留める。

## 完了条件

- 新規セーブで路地裏ステージから始まる。
- 2施設目が地下礼拝堂ではなくなる。
- 地下礼拝堂は後半施設として残る。
- 施設・アイドル・記録の解放条件が破綻していない。
- `npm run build` 成功。
- `npm run check:content` 成功。
